## 🔍 智能加载策略

### 1. 单一关键词匹配
```
用户："TypeScript 导出报错"
→ 匹配到 "TypeScript"
→ 加载 typescript-export-rules.md
```

### 2. 多关键词匹配
```
用户："如何优化这个异步函数的代码质量？"
→ 匹配到 "代码质量" + "异步"
→ 加载 coding-quality-standards.md + async-sync-safety-quick-reference.md
```

### 3. 上下文推断
```
用户："这个项目用了什么框架？"
→ 推断为技术栈问题
→ 加载 tech.md
```

### 4. 任务类型推断
```
用户："帮我生成一个需求文档"
→ 推断为文档生成任务
→ 加载 document-generation-rules.md + documentation-minimalism-rules.md
```

## 💡 使用建议

### 对于 AI（Kiro）
1. **主动识别**：扫描用户消息中的关键词
2. **智能加载**：根据映射表自动加载相关文档
3. **避免过载**：一次最多加载 2-3 个相关文档
4. **渐进式**：先用 CORE.md 回答，不够再加载详细文档

### 对于开发人员
- 无需记住所有规则文档的路径
- 只需用自然语言描述问题
- AI 会自动加载相关规则
