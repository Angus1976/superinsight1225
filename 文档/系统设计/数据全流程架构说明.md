# SuperInsight 数据全流程架构说明

**最后更新**: 2026-03-16
**维护者**: 开发团队
**状态**: 已实现（约 90%）

---

## 概述

SuperInsight 平台的数据流转采用**网状互联架构**，而非线性管道。各模块（数据同步、数据流转、数据增强、标注任务、AI 智能助手、质量管理）之间基于 RBAC 权限体系灵活互通，数据可在任意模块间循环流转、反复优化。

核心设计理念：**任何模块的输出都可以作为其他模块的输入**。

## 模块间互通关系

各模块间存在双向或多向数据引用：

### 标注任务创建 → 可从任意模块选择数据源

`TaskCreateModal` 的数据源选择支持 3 大模块 7 种数据类型：

| 分组           | 数据源类型                        | 权限要求                |
|:--------------|:---------------------------------|:-----------------------|
| 基础类型       | 文件上传、API、数据库              | —                      |
| 数据同步模块   | `ds_sources` (同步数据源)         | `dataLifecycle.create` |
| 数据流转模块   | `lc_temp_data` (临时数据)         | `dataLifecycle.create` |
| 数据流转模块   | `lc_sample` (样本库)              | `dataLifecycle.create` |
| 数据流转模块   | `lc_annotation` (标注任务)        | `annotationTask.create`|
| 数据流转模块   | `lc_enhancement` (增强数据)       | `enhancement.create`   |
| 数据流转模块   | `lc_ai_trial` (AI 试验)          | `aiTrial.view`         |
| 数据增强模块   | `aug_samples` (增强样本)          | `enhancement.create`   |

### 数据同步 → 可添加数据流转模块为数据源类型

`DataSync/Sources` 的"添加数据源"支持 5 种数据流转子类型：

| 数据源类型                  | 权限要求                |
|:--------------------------|:-----------------------|
| `lifecycle_temp_data`     | `dataLifecycle.create` |
| `lifecycle_sample`        | `dataLifecycle.create` |
| `lifecycle_enhancement`   | `enhancement.create`   |
| `lifecycle_annotation`    | `annotationTask.create`|
| `lifecycle_ai_trial`      | `aiTrial.view`         |

### 数据流转模块内部 → 子模块间状态流转

```
TempData --Review--> Samples --Create Task--> Annotation
    |                    |                        |
    |                    v                        v
    +--------------> Enhancement <---------- Results
                         |
                         v
                     AI Trial --Feedback--> Back to Enhancement/Annotation
```

### 1. 数据同步 (Data Sync)

### 2. 数据流转 (Data Lifecycle)

| 子模块   | 路由                            | API                                       | 状态流转                                        |
|:--------|:-------------------------------|:------------------------------------------|:-----------------------------------------------|
| 临时数据 | `/data-lifecycle/temp-data`    | `/api/v1/data-lifecycle/temp-data`        | raw → structured → temp_stored → under_review  |

### 6. AI 智能助手 (AI Assistant)

AI 智能助手通过权限控制的数据源访问机制，结合 LLM 和 OpenClaw 技能引擎，实现基于平台治理数据的智能分析。

```
User Chat --> AI Assistant Page (select data sources + output mode)
                  |
                  v
          Backend ChatRequest (data_source_ids + output_mode)
                  |
                  +-- build_data_context() --> Query each source
                  |       |
                  |       +-- tasks              (标注任务统计)
                  |       +-- annotation_efficiency (标注效率趋势)
                  |       +-- user_activity      (用户活跃度)
                  |       +-- data_sync          (同步状态)
                  |       +-- data_lifecycle     (流转统计)
                  |       +-- augmentation       (增强统计)
                  |       +-- quality            (质量报表)
                  |
                  +-- Inject system_prompt (merge or compare mode)
                  |
                  +-- LLM / OpenClaw --> Streaming Response
```

#### 数据源注册表（7 个预定义源）

| 数据源 ID                | 分类       | 说明                                |
|:------------------------|:----------|:------------------------------------|
| `tasks`                 | core      | 标注任务统计（总数、状态分布、完成率）   |
| `annotation_efficiency` | analytics | 标注效率（平均时间、通过率、趋势）      |
| `user_activity`         | analytics | 用户活跃度（登录、操作频次）            |
| `data_sync`             | data      | 数据同步状态（源数量、记录数、成功率）   |
| `data_lifecycle`        | data      | 数据流转统计（各阶段数据量）            |
| `augmentation`          | data      | 数据增强统计（任务数、处理量）          |
| `quality`               | analytics | 质量报表（评分分布、趋势）             |

## 数据状态枚举 (DataState)

14 个状态覆盖完整生命周期，数据不必按顺序经历所有状态：

```
raw --> structured --> temp_stored --> under_review --> approved / rejected
                                                            |
                                                    in_sample_library
                                                            |
                                      annotation_pending --> annotating --> annotated
                                                                               |
                                                             enhancing --> enhanced
                                                                               |
                                                                   trial_calculation
                                                                               |
                                                                          archived
```

数据可在 `annotated <-> enhancing` 和 `enhanced <-> trial_calculation` 之间反复迭代。

#### 完整状态定义 (`src/models/data_lifecycle.py`)

| 状态                  | 说明                 |
|:---------------------|:--------------------|
| `raw`                | 原始数据              |
| `structured`         | 已结构化              |
| `temp_stored`        | 临时存储              |
| `under_review`       | 审核中                |
| `rejected`           | 已拒绝                |
| `approved`           | 已通过                |
| `in_sample_library`  | 已入样本库            |
| `annotation_pending` | 待标注                |
| `annotating`         | 标注中                |
| `annotated`          | 已标注                |
| `enhancing`          | 增强中                |
| `enhanced`           | 已增强                |
| `trial_calculation`  | AI 试验计算中         |
| `archived`           | 已归档                |

## 权限控制体系

### 双轨权限机制

```typescript
// authStore.hasPermission(permission)
// With "."  --> DATA_LIFECYCLE_ROLE_PERMISSIONS (Data Lifecycle permissions)
// Without "." --> Permission enum (System-level permissions)
hasPermission('review.approve')    // Data Lifecycle
hasPermission('view_annotation')   // System-level
```

### 四角色权限矩阵（数据流转模块）

| 操作                | ADMIN | BUSINESS_EXPERT | ANNOTATOR | VIEWER |
|:-------------------|:-----:|:---------------:|:---------:|:------:|
| 数据创建/编辑       |  ✅   |       ✅        |    ✅     |   ❌   |
| 数据删除            |  ✅   |       ❌        |    ❌     |   ❌   |
| 归档/恢复           |  ✅   |       ✅        |    ❌     |   ❌   |
| 审核操作            |  ✅   |       ✅        |    ❌     |   ❌   |
| 增强操作            |  ✅   |       ✅        |    ❌     |   ❌   |
| 增强回滚            |  ✅   |       ❌        |    ❌     |   ❌   |
| AI 试验 (创建/运行) |  ✅   |       ✅        |    ❌     |   ❌   |
| AI 试验 (查看)      |  ✅   |       ✅        |    ✅     |   ✅   |
| 标注任务            |  ✅   |       ✅        |   部分    |   ❌   |

### 三层隔离

1. 系统级 RBAC（UserRole + Permission 枚举）
2. 数据流转权限（`module.action` 格式，动态过滤可用数据源）
3. 租户/工作空间隔离（确保多租户数据安全）

### 同步模块 RBAC (`src/sync/rbac/`)

| 组件                       | 说明                          |
|:--------------------------|:-----------------------------|
| `rbac_service.py`         | RBAC 核心服务                  |
| `permission_manager.py`   | 权限管理器                     |
| `field_access_control.py` | 字段级访问控制                  |
| `tenant_isolation.py`     | 租户隔离                       |
