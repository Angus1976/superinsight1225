## 🚀 使用指南

### 新功能开发流程
1. **需求分析**: 更新 `requirements.md`
2. **设计更新**: 更新 `design.md`  
3. **任务规划**: 更新 `tasks.md`
4. **对齐检查**: 运行对齐检查
5. **开始开发**: 按任务执行实施

### 文档维护
- **定期检查**: 每月检查文档对齐状态
- **变更同步**: 任何变更都需要同步更新三个文档
- **版本控制**: 使用Git跟踪文档变更历史
- **评审机制**: 重大变更需要团队评审

### 质量保证
- **Spec-First**: 所有开发必须先更新spec文档
- **对齐验证**: 定期运行对齐检查报告
- **测试驱动**: 基于正确性属性编写测试
- **持续集成**: 自动化测试和部署流程
