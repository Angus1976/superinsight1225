# 客户数仓对接架构说明

**最后更新**: 2026-03-16
**维护者**: 开发团队
**状态**: 已实现（约 85%）

---

## 概述

本文档说明客户数据仓库（数仓）与 SuperInsight 平台的对接架构。

核心设计理念：**旁路模式，零干预生产系统**。

SuperInsight 以只读方式从客户数仓提取数据，在平台内独立完成标注、增强、AI 试验等处理，生成的 AI 友好型数据作为客户生产数据的"旁路副本"独立存在。客户通过 API 接口或文件导出获取处理结果，无需对现有数仓做任何改造。

```
+------------------------------------------+
|     Customer Data Warehouse (Production)  |
|          READ-ONLY, NO MODIFICATION       |
+--------------------+---------------------+
                     |
                     | SELECT (read-only) / CDC (binlog/WAL)
                     v
+------------------------------------------+
|        SuperInsight Platform (Bypass)     |
|                                          |
|  Sync -> Lifecycle -> Annotation -> Aug  |
|    |                              |      |
|    +-- CDC (real-time capture)    |      |
|    +-- Datalake Management UI     |      |
|                              AI Trial    |
|                                   |      |
|          AI-Friendly Data (stored here)  |
|                   |                      |
|          +--------+--------+             |
|          v                 v             |
|     External API     File Export         |
|     (X-API-Key)     (Approval+WM)       |
|     + Webhook        + Watermark        |
+----------+-----------------+-------------+
           |                 |
           v                 v
    Client System      Data Team
    (real-time)       (offline)
```

---

## 为什么选择旁路模式

| 考量               | 旁路模式                   | 回写模式                         |
|:-------------------|:--------------------------|:---------------------------------|
| 对客户系统的影响     | 零影响，只读访问            | 需要写入权限，可能影响生产         |
| 客户 DBA 审批难度   | 低（只需 SELECT 权限）      | 高（需要 INSERT/UPDATE 权限）     |
| 数据安全风险        | 低（不触碰生产数据）         | 中（写入操作有风险）              |
| 部署复杂度          | 低（配置连接即可）           | 高（需要建表、映射、回写策略）     |
| 适用阶段            | 初期对接，快速验证           | 深度集成阶段（后续可扩展）        |

> 后续如客户需要将结果回写数仓，平台已预留输出同步能力（`OutputSyncService`），可按需开启。

## (1) 数据接入：只读提取客户数仓

### 支持的数仓/数据库类型

基于 `DataSourceType` 枚举（`src/sync/models.py`），平台支持 20+ 种数据源类型：

| 类别           | 支持的系统                              | 连接器路径                        | 协议/驱动                    |
|:--------------|:---------------------------------------|:---------------------------------|:----------------------------|
| OLAP 数仓     | ClickHouse, Apache Doris (StarRocks)   | `connectors/datalake/clickhouse.py`, `doris.py` | Native TCP / HTTP |
| 数据湖        | Hive, Iceberg, Delta Lake              | `connectors/datalake/hive.py`, `iceberg.py`, `delta_lake.py` | HiveServer2 / REST Catalog |
| 查询引擎      | Presto/Trino, SparkSQL                 | `connectors/datalake/presto_trino.py`, `spark_sql.py` | JDBC / Thrift |
| 关系型数据库   | MySQL, PostgreSQL, Oracle, SQL Server  | `connectors/database/`           | JDBC / Native              |
| 文件/对象存储  | S3, FTP/SFTP, Local File               | `connectors/file/`               | S3 API / FTP / FS          |
| API           | REST, GraphQL, SOAP, Webhook           | `connectors/api/`                | HTTP/HTTPS                 |

> 所有连接器路径前缀: `src/sync/`

#### 数仓连接器通用能力（`DatalakeBaseConnector`）

所有数仓连接器继承自 `DatalakeBaseConnector`（`connectors/datalake/base.py`），提供统一接口：

| 能力                | 方法                      | 说明                                    |
|:-------------------|:-------------------------|:---------------------------------------|
| 数据库列表浏览       | `fetch_databases()`      | 获取可用数据库列表                        |
| 表列表浏览          | `fetch_tables(db)`       | 获取表列表，含行数和大小估算               |
| 表数据预览          | `fetch_table_preview()`  | 预览表数据（默认 100 行，最大 1000 行）    |
| 自定义 SQL 查询     | `execute_query(sql)`     | 执行 SQL，含超时保护和行数限制             |
| 查询性能指标        | `get_query_metrics()`    | 总查询数、失败数、平均延迟、最近延迟        |

通用配置参数（`DatalakeConnectorConfig`）：

| 参数              | 类型    | 默认值   | 说明                |
|:-----------------|:--------|:--------|:-------------------|
| `host`           | str     | —       | 数仓主机地址         |
| `port`           | int     | —       | 端口（1-65535）      |
| `database`       | str     | —       | 数据库名             |
| `username`       | str     | —       | 认证用户名           |
| `password`       | str     | —       | 认证密码             |
| `use_ssl`        | bool    | false   | 是否启用 SSL         |
| `query_timeout`  | int     | 300     | 查询超时（秒）        |
| `max_query_rows` | int     | 100000  | 单次查询最大行数      |


### 接入模式

#### (1-A) 定时/手动提取

| 特性         | 说明                           | 对应代码                                     |
|:------------|:-------------------------------|:--------------------------------------------|
| 只读安全     | 仅需 SELECT 权限，不修改客户数据 | `connectors/`                               |
| 增量同步     | 基于时间戳或主键的增量提取        | `connectors/incremental_sync.py`            |
| 定时调度     | Cron 表达式配置自动同步频率      | `scheduler/`                                |
| 自动脱敏     | 同步过程中识别和脱敏敏感字段      | `desensitization/`                          |
| 断点续传     | 同步中断后从断点恢复             | `connectors/recovery_system.py`             |
| 连接健康监控  | 自动检测连接状态                | `connectors/database/health_monitor.py`     |
| 数据质量评分  | 同步数据自动质量评估             | `quality/`                                  |

> 代码路径前缀: `src/sync/`

同步方向枚举（`SyncDirection`）：

| 值                | 说明                    |
|:-----------------|:-----------------------|
| `pull`           | 从外部拉取到平台（默认）  |
| `push`           | 从平台推送到外部         |
| `bidirectional`  | 双向同步                |
| `local_to_cloud` | 本地到云端              |
| `cloud_to_local` | 云端到本地              |

同步频率枚举（`SyncFrequency`）：

| 值           | 说明                    |
|:------------|:-----------------------|
| `real_time` | 实时同步                |
| `scheduled` | 定时调度（Cron）         |
| `manual`    | 手动触发                |
| `on_change` | 变更触发（CDC）          |

#### (1-B) CDC 实时变更捕获

平台内置 CDC（Change Data Capture）引擎（`src/sync/cdc/database_cdc.py`），支持三种模式实时捕获数据变更：

| CDC 模式              | 适用数据库          | 原理                                    | 延迟     |
|:---------------------|:-------------------|:---------------------------------------|:--------|
| MySQL Binlog CDC     | MySQL / MariaDB    | 解析 Binlog 事件流，捕获 INSERT/UPDATE/DELETE | 秒级    |
| PostgreSQL WAL CDC   | PostgreSQL         | 解析 WAL（Write-Ahead Log）逻辑复制消息    | 秒级    |
| Polling CDC          | 任意数据库          | 定时轮询变更时间戳字段，检测新增/修改记录     | 分钟级  |

CDC 操作类型（`CDCOperation`）：

| 操作       | 说明     |
|:----------|:--------|
| `INSERT`  | 新增记录  |
| `UPDATE`  | 更新记录  |
| `DELETE`  | 删除记录  |

CDC 核心组件：

| 组件                | 说明                                              |
|:-------------------|:-------------------------------------------------|
| `BaseCDC`          | CDC 抽象基类，定义 connect/start/stop/get_changes 接口 |
| `MySQLBinlogCDC`   | MySQL Binlog 实现，支持表过滤和操作类型过滤           |
| `PostgreSQLWALCDC` | PostgreSQL WAL 实现，解析逻辑复制消息                |
| `PollingCDC`       | 轮询实现，基于时间戳字段检测变更                      |
| `CDCManager`       | CDC 实例管理器，支持注册/启停/统计                    |
| `CDCConfig`        | CDC 配置（模式、表过滤、操作过滤、批量大小等）         |

CDC 与实时同步引擎集成：

```
CDC Engine (Binlog/WAL/Polling)
        |
        | ChangeEvent (operation, table, data, timestamp)
        v
Enhanced Realtime Sync Engine
        |
        +-- realtime mode:      即时处理每条变更
        +-- near_realtime mode: 微批处理（可配置间隔）
        +-- batch mode:         定时批量处理
        |
        +-- Conflict Detection & Resolution
        |       +-- timestamp_based (默认)
        |       +-- source_wins / target_wins
        |       +-- field_merge
        |       +-- manual (人工介入)
        |       +-- business_rule (自定义规则)
        |
        v
Platform Data Storage
```


### 数仓管理 UI

平台提供专门的数仓管理前端页面（`frontend/src/pages/DataSync/Datalake/`），包含 3 个子页面：

| 页面             | 路由                              | 功能                                    |
|:----------------|:---------------------------------|:---------------------------------------|
| 数仓数据源管理   | `/data-sync/datalake/sources`    | 添加/编辑/删除数仓连接，测试连接，配置同步策略 |
| 数仓仪表盘       | `/data-sync/datalake/dashboard`  | 数仓连接状态总览，同步统计，性能指标         |
| Schema 浏览器   | `/data-sync/datalake/schema-browser` | 浏览数据库/表结构，预览数据，执行查询      |

### 配置流程

通过平台「数据同步 → 数仓管理 → 数据源管理」页面操作：

```
Step 1  选择数据源类型（如 ClickHouse / Hive / Doris 等）
Step 2  填写连接信息（主机、端口、数据库、认证凭据、SSL 选项）
Step 3  测试连接（自动验证连通性和权限）
Step 4  配置同步策略（全量 / 增量 / CDC）和调度计划（Cron 表达式）
Step 5  选择需要同步的表/视图（通过 Schema 浏览器可视化选择）
Step 6  配置数据脱敏规则（可选，自动识别 PII 字段）
```

## (2) 平台内处理：生成 AI 友好型数据

数据进入平台后，在独立环境中经过完整的 AI 数据治理流程。所有处理结果存储在 SuperInsight 平台内部，不影响客户生产数仓。

```
Synced Data --> Temp Storage --> Review --> Sample Library
                                               |
                          +--------------------+--------------------+
                          v                    v                    v
                    AI Pre-Label         Augmentation          AI Trial
                    (6+ LLM Models)     (Expand/Denoise)     (Evaluation)
                          |                    |                    |
                          +--------+-----------+                    |
                                   v                                |
                          Human Review & Label                      |
                          (Label Studio)                            |
                                   |                                |
                                   +--------------------------------+
                                                |
                                                v
                                   AI-Friendly Data (Final)
                                   +-- Annotation Results
                                   +-- Augmented Data
                                   +-- Quality Reports (Ragas)
                                   +-- AI Trial Results
                                   +-- Data Lineage Records
```

| 处理阶段   | 说明                                                       |
|:----------|:----------------------------------------------------------|
| 数据流转   | 临时存储 → 审核 → 入库，全程版本控制和审计日志                  |
| AI 预标注  | 集成 6+ LLM 提供商（Ollama/ChatGLM/Baidu/Hunyuan/Alibaba/HuggingFace）智能预标注 |
| 人工标注   | 基于 Label Studio 的人机协作，支持文本分类、实体识别、关系抽取等 |
| 质量评估   | 基于 Ragas 框架的语义质量评估，自动评分和改进建议               |
| 数据增强   | 数据扩充、降噪、特征提取、归一化，支持 LLM 驱动的智能增强       |
| AI 试验   | 模型效果验证，结果可反馈回标注/增强环节迭代优化                   |
| 数据血缘   | 全链路血缘追踪，记录数据从源到目标的完整流转路径                  |

## (3) 结果获取：两种输出路径

### (3-A) 外部 API — 推荐，实时访问

客户业务系统通过 RESTful API 实时查询处理结果，适合程序化集成场景。

#### API 端点（`src/sync/gateway/external_data_router.py`）

| 端点                                      | 数据类型     | 说明                       |
|:-----------------------------------------|:------------|:--------------------------|
| `GET /api/v1/external/annotations`       | 标注结果     | 文本分类、实体识别等标注数据 |
| `GET /api/v1/external/augmented-data`    | 增强数据     | 经过扩充、降噪处理的数据     |
| `GET /api/v1/external/quality-reports`   | 质量报告     | 数据质量评分和分析           |
| `GET /api/v1/external/experiments`       | AI 试验结果  | 模型评估和效果验证数据       |

#### 认证与安全

| 项目           | 说明                                                          |
|:--------------|:-------------------------------------------------------------|
| 认证方式       | 请求头 `X-API-Key`，密钥格式 `sk_` 前缀                        |
| 密钥安全       | SHA-256 哈希存储（`key_hash`），完整密钥仅创建时可见一次，仅保留 `key_prefix` 用于识别 |
| 权限范围       | 通过 `scopes`（JSONB）控制可访问的端点（如仅允许访问标注结果）     |
| 速率限制       | 按密钥独立配置 `rate_limit_per_minute`（默认 60）/ `rate_limit_per_day`（默认 10000），超限返回 `429` + `Retry-After` |
| 密钥生命周期   | `active` ↔ `disabled` 可切换，`revoked` 为终态不可恢复          |
| 过期控制       | `expires_at` 可选过期时间                                      |
| 调用审计       | 每次调用记录端点、状态码、响应时间（`APICallLogModel`）           |
| 用量统计       | `total_calls` + `last_used_at` 自动更新                       |

#### Webhook 通知（新增）

API 密钥支持配置 Webhook，当数据处理完成时主动通知客户系统：

| 字段              | 说明                                    |
|:-----------------|:---------------------------------------|
| `webhook_url`    | 回调地址（HTTPS），接收处理完成通知        |
| `webhook_secret` | 签名密钥，用于验证回调请求的真实性         |
| `webhook_events` | 订阅的事件类型（如标注完成、增强完成等）    |

> Webhook 配置存储在 `APIKeyModel.webhook_config`（JSONB），通过 API 管理页面（`/data-sync/api-management`）配置。

#### 智能服务引擎扩展

API 密钥还支持智能服务引擎能力：

| 字段                      | 说明                                              |
|:-------------------------|:-------------------------------------------------|
| `allowed_request_types`  | 允许的请求类型：`query`, `chat`, `decision`, `skill` |
| `skill_whitelist`        | 允许调用的技能白名单                                 |

#### 响应格式

统一 JSON 分页响应，支持 `page`、`page_size`、`sort_by`、`fields`（字段筛选）参数：

```json
{
    "total": 1250,
    "page": 1,
    "page_size": 50,
    "items": [...]
}
```

## 数据血缘追踪

平台内置数据血缘追踪引擎（`src/sync/lineage/lineage_tracker.py`），记录数据从客户数仓到最终输出的完整流转路径。

### 血缘图模型

| 组件              | 说明                                              |
|:-----------------|:-------------------------------------------------|
| `LineageNode`    | 血缘节点（数据源、转换、目标等），含类型、名称、元数据 |
| `LineageEdge`    | 血缘边（数据流向），含源节点、目标节点、转换类型      |
| `LineageRecord`  | 记录级血缘，追踪单条数据的完整流转路径               |
| `LineageGraph`   | 血缘图，支持上下游遍历、路径查找、可视化导出          |

### 节点类型（`LineageNodeType`）

| 类型            | 说明                    |
|:---------------|:-----------------------|
| `source`       | 数据源（客户数仓）       |
| `transformation`| 数据转换节点            |
| `target`       | 目标节点（输出）         |

### 核心能力

| 能力                | 方法                          | 说明                                    |
|:-------------------|:-----------------------------|:---------------------------------------|
| 注册数据源          | `register_source()`          | 注册外部数据源节点                        |
| 注册目标            | `register_target()`          | 注册输出目标节点                          |
| 注册转换            | `register_transformation()`  | 注册数据转换节点                          |
| 添加数据流          | `add_data_flow()`            | 建立节点间的数据流向关系                   |
| 记录级追踪          | `track_record()`             | 追踪单条记录的血缘                        |
| 批量追踪            | `track_batch()`              | 批量追踪记录血缘                          |
| 下游影响分析        | `get_downstream_impact()`    | 分析某节点变更对下游的影响范围              |
| 上游溯源            | `get_upstream_sources()`     | 追溯数据的上游来源                        |
| 数据流路径          | `get_data_flow_path()`       | 获取两个节点间的数据流转路径               |
| 可视化数据          | `get_visualization_data()`   | 导出血缘图的可视化数据（节点 + 边）         |

## 典型对接场景

### 场景一：数仓数据标注 + API 查询

```
Customer ClickHouse
        |
        | (1) Scheduled incremental sync (daily, read-only)
        v
SuperInsight (bypass processing)
        |
        | (2) AI pre-label (Ollama/ChatGLM) + human review
        v
Client System
        |
        | (3) GET /api/v1/external/annotations
        |     X-API-Key: sk_xxxx
        |
        +-- Receive JSON annotation results
        +-- Webhook notification on completion (optional)
```

### 场景二：CDC 实时同步 + 增强数据导出

```
Customer MySQL (Production)
        |
        | (1) MySQL Binlog CDC (real-time change capture)
        v
SuperInsight (real-time processing)
        |
        | (2) Auto desensitization + augmentation + quality check
        v
Data Team
        |
        | (3) Submit export request (select augmented dataset)
        v
SuperInsight Export Approval
        |
        | (4) Approved, generate watermarked file
        v
Download CSV/JSON
        |
        +-- Import into internal analysis system
        +-- Full data lineage available for audit
```

### 场景三：数据湖对接 + Schema 浏览

```
Customer Hive / Iceberg Data Lake
        |
        | (1) Connect via Datalake Management UI
        |     Browse databases → tables → preview data
        v
SuperInsight Schema Browser
        |
        | (2) Select tables, configure sync strategy
        |     Full sync → then incremental via timestamp
        v
SuperInsight Processing Pipeline
        |
        | (3) Lifecycle → Label → Augment → AI Trial
        v
Results available via API + Export
```

## 后端子模块总览

数仓对接涉及的所有后端子模块（`src/sync/`）：

| 子模块           | 路径                        | 职责                                           |

| orchestrator    | `orchestrator/`            | 同步编排（事件管理、流程编排）                     |
| gateway         | `gateway/`                 | API 网关（认证、限流、外部数据路由、安全）          |
| push            | `push/`                    | 输出同步（推送、增量推送、目标管理、结果校验）       |
| export_control  | `export_control/`          | 导出控制（审批流程、水印、权限、监控）              |
