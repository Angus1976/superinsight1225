# 结构化进度追踪优化

**创建时间**: 2026-03-05  
**优化目标**: 解决非结构化数据结构化过程卡住的问题，提供实时进度追踪

---

## 📋 问题分析

### 原有问题

1. **LLM 超时配置不合理**
   - 默认超时 300 秒（5 分钟）过长
   - 重试机制导致总等待时间可达 15 分钟
   - 即使数据量少也可能卡住

2. **错误重试策略不当**
   - 所有未知错误都会重试
   - JSON 解析错误、格式错误也会重试
   - 导致无效重试浪费时间

3. **缺少进度反馈**
   - 任务状态粗粒度，无法知道具体进展
   - 用户不知道是在等待还是真的卡住了
   - 没有预估剩余时间

---

## ✅ 优化方案

### 1. 降低 LLM 超时时间

**修改文件**: `src/ai/llm_schemas.py`

```python
# 从 300 秒降低到 60 秒
timeout: int = Field(default=60, ge=1, le=600)
```

**效果**: 单次 LLM 调用最多等待 60 秒，总重试时间从 15 分钟降至 3 分钟

### 2. 优化错误重试策略

**修改文件**: `src/ai/retry.py`

**改进**:
- 不重试 JSON 解析错误
- 不重试 instructor 解析错误
- 不重试 ValidationError
- 未知错误默认不重试（fail fast）

**效果**: 避免无效重试，快速失败并报告真实错误

### 3. 添加详细进度追踪

**新增文件**: `src/services/progress_tracker.py`

**功能**:
- 6 个步骤的详细进度追踪
- 每个步骤的进度百分比
- 实时消息和详细信息
- 已用时间和预估剩余时间

**数据结构**:

### 方法 3: Python 代码集成

```python
from src.database.connection import db_manager
from src.models.structuring import StructuringJob

# 查询任务进度
with db_manager.get_session() as session:
    job = session.query(StructuringJob).filter_by(id=job_id).first()
    
    if job.progress_info:
        progress = job.progress_info
        print(f"总体进度: {progress['overall_progress_percent']:.1f}%")
        print(f"当前步骤: {progress['current_step']}/{progress['total_steps']}")
        print(f"已用时间: {progress['elapsed_seconds']}s")
        
        for step in progress['steps']:
            print(f"步骤 {step['step_number']}: {step['step_name']}")
            print(f"  状态: {step['status']}")
            print(f"  进度: {step['progress_percent']:.1f}%")
            print(f"  消息: {step['message']}")
```

## 🔧 故障排查

### 问题 1: 进度信息为空

**原因**: 旧版本任务或数据库未迁移

**解决**:
```bash
# 运行数据库迁移
alembic upgrade head
```

### 问题 2: 进度不更新

**原因**: Celery worker 未重启

**解决**:
```bash
# 重启 Celery worker
celery -A src.sync.async_queue.celery_integration worker --loglevel=info
```

### 问题 3: 任务仍然卡住

**检查步骤**:
1. 查看日志确认是哪个步骤卡住
2. 检查 LLM 服务是否正常
3. 检查网络连接
4. 查看错误消息

**日志位置**:
```bash
# 查看 Celery worker 日志
tail -f logs/celery_worker.log

# 查看应用日志
tail -f logs/app.log
```
