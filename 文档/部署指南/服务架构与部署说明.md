**最后更新**: 2026-02-27  
**维护者**: 开发团队

## 系统架构概览

SuperInsight 是一个企业级 AI 数据治理与智能标注平台，采用微服务架构，通过 Docker Compose 编排 10 个服务容器。

```
                    ┌─────────────────────────────────────────────┐
                    │              用户浏览器                       │
                    └──────┬──────────────────┬───────────────────┘
                           │                  │
                    :5173 (前端)         :8000 (API)
                           │                  │
                ┌──────────▼──┐    ┌──────────▼──────────┐
                │  Frontend   │    │   App (FastAPI)      │
                │  React/Vite │───▶│   核心业务服务        │
                └─────────────┘    └──┬──┬──┬──┬──┬──┬───┘
                                     │  │  │  │  │  │
              ┌──────────────────────┘  │  │  │  │  └──────────────┐
              │           ┌─────────────┘  │  │  └────────┐        │
              ▼           ▼                │  │           ▼        ▼
        ┌──────────┐ ┌─────────┐          │  │    ┌──────────┐ ┌────────┐
        │PostgreSQL│ │  Redis  │          │  │    │  Ollama  │ │Argilla │
        │  数据库   │ │  缓存   │          │  │    │ 本地 LLM │ │AI 评估 │
        └──────────┘ └─────────┘          │  │    └──────────┘ └───┬────┘
                                          │  │                     │
                                          ▼  ▼                     ▼
                                   ┌────────────┐          ┌──────────────┐
                                   │Label Studio│          │Elasticsearch │
                                   │  标注引擎   │          │  搜索引擎     │
                                   └────────────┘          └──────────────┘

        ┌──────────────── 监控层 ────────────────┐
        │  Prometheus (:9090) → Grafana (:3001)  │
        └────────────────────────────────────────┘
```

## 服务详细说明

### 核心业务服务

#### 1. App — 后端 API 服务

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-app` |
| 端口 | `8000` |
| 镜像 | 自定义构建（`Dockerfile`，Python 3.11-slim） |
| 必需 | 是 |

平台的核心服务，基于 FastAPI 构建。负责：
- RESTful API 提供所有业务接口（`/api/v1/{domain}`）
- 用户认证与 RBAC 权限控制（JWT + bcrypt）
- 数据提取、标注任务管理、质量评估
- AI 模型集成调度（Ollama、OpenAI）
- 数据脱敏中间件
- 多租户隔离
- Prometheus 指标暴露

依赖：PostgreSQL、Redis、Label Studio、Argilla、Ollama

#### 2. Frontend — 前端应用

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-frontend` |
| 端口 | `5173` |
| 镜像 | 自定义构建（`frontend/Dockerfile`） |
| 必需 | 是 |

基于 React 19 + TypeScript + Vite 7 构建的 SPA 应用。负责：
- 数据管理控制台（Ant Design Pro）
- 标注任务管理界面
- 系统管理后台（租户、权限、配额）
- 多语言支持（中文/英文）
- 实时数据可视化

通过 `VITE_API_BASE_URL` 连接后端 API。开发模式下挂载 `src/` 和 `public/` 目录实现热更新。

### 数据存储服务

#### 3. PostgreSQL — 主数据库

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-postgres` |
| 端口 | `5432` |
| 镜像 | `postgres:15-alpine` |
| 必需 | 是 |

系统的主要持久化存储，承载：
- 用户、租户、权限等核心业务数据
- 标注任务和标注结果
- 审计日志和操作记录
- Argilla 的数据存储（独立的 `argilla` 数据库）
- JSONB 字段支持灵活 schema

默认凭据：用户 `superinsight`，密码 `password`，数据库 `superinsight`。数据通过 Docker Volume `postgres_data` 持久化。使用 Alembic 管理数据库迁移。

#### 4. Redis — 缓存与消息队列

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-redis` |
| 端口 | `6379` |
| 镜像 | `redis:7-alpine` |
| 必需 | 是 |

高性能缓存和消息中间件，负责：
- API 响应缓存和会话管理
- Celery 任务队列后端
- WebSocket Pub/Sub 消息分发
- AI 标注建议缓存
- 速率限制计数器

启用 AOF 持久化（`--appendonly yes`），数据通过 `redis_data` Volume 持久化。

#### 5. Elasticsearch — 全文搜索引擎

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-elasticsearch` |
| 端口 | `9200` |
| 镜像 | `docker.elastic.co/elasticsearch/elasticsearch:8.5.0` |
| 必需 | 是（Argilla 依赖） |

为 Argilla 提供底层搜索和索引能力：
- 标注数据的全文检索
- 数据集的语义搜索
- 聚合分析和统计

单节点模式运行，关闭 xpack 安全认证。JVM 堆内存限制 512MB。数据通过 `elasticsearch_data` Volume 持久化。

### AI 与标注服务

#### 6. Label Studio — 标注引擎

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-label-studio` |
| 端口 | `8080` |
| 镜像 | 自定义构建（`deploy/label-studio/Dockerfile`） |
| 必需 | 是 |

核心标注引擎，经过定制化构建，包含：
- SSO 单点登录集成（可通过环境变量开关）
- 自定义品牌样式（`branding.css`）
- 中文语言支持（Django i18n，`zh-hans`）
- 自定义 URL 路由和设置覆盖

支持多种标注类型：文本分类、NER、图像标注、音频标注等。通过 API Token 或 JWT 与后端 App 服务通信。

#### 7. Argilla — AI 辅助评估

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-argilla` |
| 端口 | `6900` |
| 镜像 | `argilla/argilla-server:latest` |
| 必需 | 否（增强功能） |

AI 辅助数据标注和评估平台，负责：
- LLM 输出质量评估
- 人机协作标注工作流
- 标注数据质量反馈循环
- 与 Ragas 框架集成的语义质量评估

依赖 Elasticsearch 和 PostgreSQL（使用独立的 `argilla` 数据库）。以 root 用户运行以解决权限问题。

#### 8. Ollama — 本地 LLM 服务

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-ollama` |
| 端口 | `11434` |
| 镜像 | `ollama/ollama:latest` |
| 必需 | 否（可用外部 LLM API 替代） |

本地大语言模型推理服务，提供：
- AI 预标注（自动标注建议）
- Text-to-SQL 自然语言查询
- 数据分类和实体识别
- 离线环境下的 AI 能力

模型数据通过 `ollama_data` Volume 持久化。首次使用需手动拉取模型：
```bash
docker compose exec ollama ollama pull llama2
```

如有 GPU 可用，建议在 `docker-compose.yml` 中添加 GPU 资源配置以加速推理。

### 监控服务

#### 9. Prometheus — 指标采集

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-prometheus` |
| 端口 | `9090` |
| 镜像 | `prom/prometheus:latest` |
| 必需 | 否（生产环境推荐） |

时序数据库和指标采集系统，负责：
- 从 App 服务抓取业务指标（API 延迟、请求量、错误率）
- 数据库连接池状态监控
- Redis 缓存命中率
- 系统资源使用情况

配置文件挂载自项目根目录的 `prometheus.yml`。

#### 10. Grafana — 监控仪表板

| 项目 | 说明 |
|------|------|
| 容器名 | `superinsight-grafana` |
| 端口 | `3001`（映射容器内 3000） |
| 镜像 | `grafana/grafana:latest` |
| 必需 | 否（生产环境推荐） |

可视化监控仪表板，提供：
- 系统概览 Dashboard
- API 性能监控
- 数据库性能分析
- 业务指标可视化
- 告警规则配置

默认凭据：用户 `admin`，密码 `admin`。数据源自动连接 Prometheus。

## 部署配置

### Compose 文件说明

| 文件 | 用途 | 服务数 |
|------|------|--------|
| `docker-compose.yml` | 完整栈（开发/测试） | 10 个 |
| `docker-compose.minimal.yml` | 最小栈（轻量开发） | 4 个（App + Frontend + PostgreSQL + Redis） |
| `docker-compose.ai-integration.yml` | AI 应用集成扩展 | +2 个（OpenClaw Gateway + Agent） |

### 网络架构

所有服务运行在同一个 Docker Bridge 网络 `superinsight_network` 中，服务间通过容器名互相访问（如 `postgres:5432`、`redis:6379`）。

对外暴露的端口：

| 端口 | 服务 | 用途 |
|------|------|------|
| 5173 | Frontend | 前端界面 |
| 8000 | App | 后端 API + Swagger 文档 |
| 8080 | Label Studio | 标注平台 |
| 6900 | Argilla | AI 评估平台 |
| 11434 | Ollama | LLM API |
| 9090 | Prometheus | 监控指标 |
| 3001 | Grafana | 监控仪表板 |
| 5432 | PostgreSQL | 数据库（开发用） |
| 6379 | Redis | 缓存（开发用） |
| 9200 | Elasticsearch | 搜索（开发用） |

### 数据持久化

所有有状态服务通过 Docker Named Volume 持久化数据：

| Volume | 服务 | 数据内容 |
|--------|------|---------|
| `postgres_data` | PostgreSQL | 数据库文件 |
| `redis_data` | Redis | AOF 持久化数据 |
| `label_studio_data` | Label Studio | 标注数据和上传文件 |
| `argilla_data` | Argilla | Argilla 工作数据 |
| `elasticsearch_data` | Elasticsearch | 索引数据 |
| `ollama_data` | Ollama | 下载的 LLM 模型 |
| `prometheus_data` | Prometheus | 时序指标数据 |
| `grafana_data` | Grafana | 仪表板配置和数据 |

## 快速启动

### 完整栈启动

```bash
# 克隆项目
git clone <repo-url>
cd superdata

# 配置环境变量
cp .env.example .env
# 编辑 .env 修改必要配置

# 启动所有服务
docker compose up -d --build

# 检查服务状态
docker compose ps

# 查看日志
docker compose logs -f app
```

### 最小栈启动（轻量开发）

```bash
docker compose -f docker-compose.minimal.yml up -d --build
```

只启动 App、Frontend、PostgreSQL、Redis 四个核心服务，适合前后端开发调试。

### AI 集成扩展

```bash
docker compose -f docker-compose.yml -f docker-compose.ai-integration.yml up -d
```

在完整栈基础上增加 OpenClaw Gateway 和 Agent，支持多渠道 AI 助手（WhatsApp、Telegram、Slack 等）。

## 服务依赖关系

```
PostgreSQL ◄── App ──► Redis
    ▲              │
    │              ├──► Label Studio
    │              ├──► Argilla ──► Elasticsearch
    │              └──► Ollama
    │
    └── Argilla

Frontend ──► App

Grafana ──► Prometheus ──► App
```

启动顺序（Docker Compose 自动处理）：
1. PostgreSQL、Redis、Elasticsearch（无依赖）
2. Ollama（无依赖）
3. Label Studio、Argilla（依赖数据库和搜索）
4. App（依赖所有后端服务）
5. Frontend（依赖 App）
6. Prometheus（无依赖，但需要 App 提供指标）
7. Grafana（依赖 Prometheus）

## 环境变量

关键环境变量（完整列表见 `.env.example`）：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `APP_ENV` | `development` | 运行环境 |
| `DATABASE_URL` | `postgresql://superinsight:password@postgres:5432/superinsight` | 数据库连接 |
| `REDIS_URL` | `redis://redis:6379/0` | Redis 连接 |
| `LABEL_STUDIO_URL` | `http://label-studio:8080` | Label Studio 地址 |
| `ARGILLA_URL` | `http://argilla:6900` | Argilla 地址 |
| `OLLAMA_BASE_URL` | `http://ollama:11434` | Ollama 地址 |
| `JWT_SECRET_KEY` | 需自行设置 | JWT 签名密钥 |
| `CORS_ORIGINS` | `http://localhost:3000,http://localhost:5173` | 允许的跨域来源 |

## 生产环境注意事项

1. 修改所有默认密码（PostgreSQL、Grafana、Label Studio、JWT Secret）
2. 关闭开发端口（5432、6379、9200 不应对外暴露）
3. 配置 Nginx 反向代理 + HTTPS
4. 为 Ollama 配置 GPU 资源
5. 调整 Elasticsearch JVM 堆内存（生产建议 2GB+）
6. 配置数据库定期备份
7. 设置 Grafana 告警规则
