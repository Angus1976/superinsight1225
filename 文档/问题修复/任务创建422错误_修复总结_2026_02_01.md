## 问题概述

用户在创建新任务时遇到 422 Unprocessable Entity 错误，后端日志显示只收到 `assignee_id` 字段，缺少必需的 `name` 字段。

### 关键发现

1. **不是浏览器缓存问题**: 用户已清除缓存并在无痕模式测试，问题仍存在
2. **多步骤表单问题**: 表单使用 4 步向导，可能存在状态管理问题
3. **字段丢失**: 前端表单中填写的字段在提交时丢失

---

在提交前显式检查所有必需字段：

```typescript
// Validate required fields explicitly
if (!values.name) {
  message.error(t('taskNameRequired'));
  setCurrentStep(0); // Go back to step 0 where name field is

}

if (!values.priority) {
  message.error(t('priorityRequired'));
  setCurrentStep(0);

  return;
}
```

**目的**: 
- 在客户端就拦截不完整的数据
- 自动跳转到缺失字段的步骤
- 提供清晰的错误提示

## 可能的根本原因

### 假设 1: 表单状态未正确保存

**症状**: 用户填写的字段在步骤切换时丢失

**验证方法**: 
1. 在步骤 0 填写字段
2. 点击"下一步"
3. 点击"上一步"返回
4. 检查字段值是否保留

**如果字段丢失**: 需要检查 Ant Design Form 的配置

### 假设 2: 异步验证时机问题

**症状**: 快速点击"下一步"时，验证未完成就进入下一步

**验证方法**: 
1. 快速连续点击"下一步"
2. 观察是否能跳过验证

**当前状态**: 代码中已使用 `await form.validateFields()`，应该不会有这个问题

### 假设 3: React 状态更新时机

**症状**: 表单值更新但 React 状态未同步

**验证方法**: 
1. 在控制台查看 `Form values before submission`
2. 对比用户实际填写的值

**如果不一致**: 需要检查表单字段的 `name` 属性和 `Form.Item` 配置
