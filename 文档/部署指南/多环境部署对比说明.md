# 多环境部署对比说明

**最后更新**: 2026-03-24
**适用版本**: SuperInsight v0.85+

---

## 三套部署配置

| 配置 | 用途 | 入口文件 |
|------|------|----------|
| `docker-compose.yml` | 日常开发（热更新） | `.env` |
| `docker-compose.prod.yml` | 本地生产验证 | `.env.prod.local` |
| Sealos 应用管理 UI | 云端生产 | `deploy/sealos/.env.sealos` |

---

## 核心差异对比

| 维度 | 开发模式 | 本地生产验证 | Sealos 生产 |
|------|---------|-------------|------------|
| 前端 | Vite dev server (5173) | nginx (80) | nginx (80) |
| 前端代码 | 挂载 volume 热更新 | bake 进镜像 | bake 进镜像 |
| 后端代码 | 挂载 volume 热更新 | bake 进镜像 | bake 进镜像 |
| 后端 Dockerfile | `./Dockerfile` | `deploy/sealos/Dockerfile.backend` | 同左 |
| 前端 Dockerfile | `./frontend/Dockerfile` | `deploy/local/Dockerfile.frontend` | `deploy/sealos/Dockerfile.frontend` |
| nginx upstream | 无（直连后端） | `backend:8000`（Compose 服务名） | K8s FQDN（Sealos 内网） |
| 数据库 | 宿主机 PG 或容器 PG | 容器 PG (pgvector) | Sealos 托管 PG |
| Label Studio | `deploy/label-studio/Dockerfile` | `deploy/sealos/Dockerfile.labelstudio` | 同左 |
| ES | 原版镜像 | 原版镜像 | 自定义镜像（PVC 权限） |
| LLM | Ollama 容器 | Ollama 容器 | 外部 API（DeepSeek/通义千问） |
| 监控 | Prometheus + Grafana | 无 | 无 |
| Celery | compose command 覆盖 | compose command 覆盖 | 专用 Dockerfile.celery |

---

## 共享部分（不因环境而异）

- 应用代码：`src/`、`frontend/src/` — 完全相同
- 环境变量名：统一命名，值按环境不同
- 后端 Dockerfile：`deploy/sealos/Dockerfile.backend` 本地生产和 Sealos 共用
- LS Dockerfile：`deploy/sealos/Dockerfile.labelstudio` 本地生产和 Sealos 共用（含 SSO + 品牌化 + PVC 修复）
- JWT 认证：所有环境统一从 `JWT_SECRET_KEY` 环境变量读取

---

## 仅因环境不同的配置

### nginx.conf

| 文件 | 用途 | upstream 地址 |
|------|------|--------------|
| `deploy/local/nginx.conf` | 本地生产验证 | `http://backend:8000`（Docker DNS） |
| `deploy/sealos/nginx.conf` | Sealos 生产 | K8s FQDN |

差异原因：Docker Compose 用服务名解析，K8s 用 FQDN。resolver 也不同（`127.0.0.11` vs `kube-dns`）。

### VITE_LABEL_STUDIO_URL

| 环境 | 值 | 说明 |
|------|-----|------|
| 开发 | `http://localhost:8080` | 直连本地 LS |
| 本地生产 | `http://localhost:8080` | 同上 |
| Sealos | LS 外网域名 | Sealos 分配 |

此变量在前端构建时 bake 进产物，运行时不可更改。

### LABEL_STUDIO_URL（后端）

| 环境 | 值 |
|------|-----|
| 开发 | `http://label-studio:8080` |
| 本地生产 | `http://label-studio:8080` |
| Sealos | LS 内网 FQDN |

## 使用方式

### 日常开发
```bash
docker compose up
# 访问 http://localhost:5173
```

### 本地生产验证（推送 Sealos 前）
```bash
docker compose -f docker-compose.prod.yml --env-file .env.prod.local up --build
# 访问 http://localhost
```

### Sealos 部署
```bash
# 构建并推送镜像（以后端为例）
docker build --platform linux/amd64 -f deploy/sealos/Dockerfile.backend -t angus888/superinsight-backend:sealos .
docker push angus888/superinsight-backend:sealos
# 在 Sealos 应用管理中重启对应应用
```

## 文件清单

```
docker-compose.yml              # 开发用
docker-compose.prod.yml         # 本地生产验证
.env                            # 开发环境变量
.env.prod.local                 # 本地生产环境变量
deploy/
├── local/
│   ├── Dockerfile.frontend     # 本地生产前端（nginx + 本地 upstream）
│   └── nginx.conf              # 本地 nginx 配置
├── sealos/
│   ├── Dockerfile.backend      # 后端（本地生产 + Sealos 共用）
│   ├── Dockerfile.frontend     # Sealos 前端（nginx + K8s upstream）
│   ├── Dockerfile.celery       # Sealos Celery（写死 CMD）
│   ├── Dockerfile.labelstudio  # LS（本地生产 + Sealos 共用）
│   ├── Dockerfile.elasticsearch # ES PVC 权限修复（仅 Sealos）
│   ├── nginx.conf              # Sealos nginx 配置
│   └── .env.sealos             # Sealos 环境变量参考
└── label-studio/               # LS SSO + 品牌化源文件
```
