# 文档最小化规范

**版本**: 1.0  
**状态**: ✅ Active  
**优先级**: HIGH

## 核心原则

**信息爆炸 = 无用信息**

只生成必要的文档。除需求外，其他文档都要尽可能简化或不生成。

## 文档分类

### ✅ 必须生成

1. **requirements.md** - 需求文档（完整）
   - 用户故事
   - 验收标准
   - 优先级

2. **design.md** - 设计文档（精简）
   - 架构决策
   - 关键接口
   - 数据模型

3. **tasks.md** - 任务列表（精简）
   - 任务分解
   - 时间估计
   - 依赖关系

### ❌ 禁止生成

- 诊断报告
- 执行报告
- 实现总结
- 测试指南（除非特别需要）
- 测试检查清单
- 测试准备文档
- 快速启动指南
- 完成总结
- 资源索引
- README（除非项目级）

## 规则

### 规则 1: 一个文档一个目的

**❌ 错误**:
```
DIAGNOSIS_REPORT.md (诊断)
IMPLEMENTATION_SUMMARY.md (实现)
EXECUTION_REPORT.md (执行)
MANUAL_TESTING_GUIDE.md (测试)
MANUAL_TESTING_CHECKLIST.md (检查)
QUICK_START_MANUAL_TESTING.md (快速启动)
TASK_5_1_COMPLETION_SUMMARY.md (完成)
```

**✅ 正确**:
```
requirements.md (需求)
design.md (设计)
tasks.md (任务)
```

### 规则 2: 文档大小限制

- **requirements.md**: < 5000 字
- **design.md**: < 3000 字
- **tasks.md**: < 2000 字

超过限制则拆分或删除非必要内容。

### 规则 3: 内容精简

**删除**:
- 重复信息
- 冗余解释
- 过度详细的步骤
- 多个版本的同一内容

**保留**:
- 核心信息
- 关键决策
- 必要的上下文

### 规则 4: 测试文档

**不生成**:
- 测试指南
- 测试检查清单
- 测试报告
- 测试总结

**替代方案**:
- 在 tasks.md 中简要说明测试方法
- 在代码注释中说明测试用例
- 在 CI/CD 中自动生成测试报告

### 规则 5: 实现文档

**不生成**:
- 诊断报告
- 执行报告
- 实现总结
- 完成总结

**替代方案**:
- 在 tasks.md 中更新进度
- 在 Git commit 中记录变更
- 在代码中添加注释

## 执行检查

### 创建新文档前

- [ ] 这个文档是必须的吗？
- [ ] 是否已有类似文档？
- [ ] 能否合并到现有文档？
- [ ] 内容是否超过 3000 字？

### 创建新文档时

- [ ] 文档名称清晰简洁
- [ ] 内容精简无冗余
- [ ] 没有重复信息
- [ ] 大小在限制内

### 审查现有文档

- [ ] 是否有冗余文档？
- [ ] 是否有重复内容？
- [ ] 是否超过大小限制？
- [ ] 是否有过时信息？

## 违规处理

**发现冗余文档**:
1. 立即删除
2. 合并到必要文档
3. 更新相关引用

**发现超大文档**:
1. 删除非必要内容
2. 拆分为多个小文档（如必要）
3. 保留核心信息
