# 任务创建 422 错误诊断与修复

**日期**: 2026-02-01  
**问题**: 创建新任务时返回 422 Unprocessable Entity 错误  
**状态**: ✅ 已修复并添加调试日志

---

## 问题描述

用户在创建新任务时遇到 422 错误，即使在清除浏览器缓存和使用无痕模式后问题仍然存在。

### 错误信息

```
POST /api/tasks HTTP/1.1 422 Unprocessable Entity
```

### 后端日志

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "name"],
      "msg": "Field required",
      "input": {"assignee_id": "user1"}
    }
  ]
}
```

**关键发现**: 后端只收到了 `assignee_id` 字段，但缺少必需的 `name` 字段。

---

## 根本原因分析

### 1. 不是浏览器缓存问题

- 用户已清除 4 周浏览器缓存
- 在无痕模式下测试仍然失败
- 说明这是代码逻辑问题，不是缓存问题

### 2. 多步骤表单验证问题

**问题**: 表单使用多步骤向导（4 个步骤），每个步骤只验证当前步骤的字段。

**步骤分布**:
- **步骤 0**: 基本信息（name, description, priority, due_date, tags）
- **步骤 1**: 数据源（data_source_type, data_source_id）
- **步骤 2**: 标注配置（annotation_type）
- **步骤 3**: 分配配置（assignee_id）

**问题所在**:
1. 用户在步骤 0 填写 `name` 字段
2. 点击"下一步"时，只验证步骤 0 的字段
3. 用户继续到步骤 3，点击"创建任务"
4. `handleSubmit` 调用 `form.validateFields()` 验证所有字段
5. **但是**，如果用户在某个步骤修改了字段但没有触发验证，或者表单状态没有正确保存，就会导致某些字段丢失

### 3. 可能的触发场景

- 用户快速点击"下一步"，表单状态未及时更新
- 用户在某个步骤返回修改，但没有重新验证
- 表单初始值设置不正确
- React 状态更新时机问题

---

## 修复方案

### 修复 1: 添加调试日志

在 `handleSubmit` 函数中添加详细的调试日志，帮助诊断问题：

```typescript
const handleSubmit = async () => {
  try {
    // Validate all fields before submission
    const values = await form.validateFields();
    
    // Debug logging to see what values we're getting
    console.log('Form values before submission:', values);
    
    // Validate required fields explicitly
    if (!values.name) {
      message.error(t('taskNameRequired'));
      setCurrentStep(0); // Go back to step 0 where name field is
      return;
    }
    
    if (!values.priority) {
      message.error(t('priorityRequired'));
      setCurrentStep(0);
      return;
    }
    
    if (!values.annotation_type) {
      message.error(t('annotationTypeRequired'));
      setCurrentStep(2);
      return;
    }
    
    if (!values.assignee_id) {
      message.error(t('assigneeRequired'));
      setCurrentStep(3);
      return;
    }
    
    // ... rest of the code
    
    console.log('Payload being sent to API:', payload);
    
    await createTask.mutateAsync(payload);
    // ...
  } catch (error) {
    console.error('Error creating task:', error);
    // ...
  }
};
```

### 修复 2: 显式验证必需字段

在提交前显式检查所有必需字段，如果缺失则：
1. 显示错误消息
2. 自动跳转到包含该字段的步骤
3. 阻止提交

## 可能的问题场景

### 场景 1: 表单初始值问题

**症状**: 某些字段始终为空

**检查**:
```typescript
// 在 TaskCreateModal.tsx 中检查 initialValues
<Form
  form={form}
  layout="vertical"
  initialValues={{
    priority: 'medium',
    annotation_type: 'text_classification',
    data_source_type: 'file',
    auto_assignment: false,
    notification_enabled: true,
    deadline_reminder: true,
  }}
>
```

**问题**: `name` 字段没有初始值，如果用户没有输入就会是 `undefined`

**解决**: 确保用户必须输入 `name` 字段（已通过 `required` 规则实现）

### 场景 2: 步骤导航问题

**症状**: 用户返回上一步修改后，字段值丢失

**检查**: 表单是否使用 `preserveOnUnmount` 保持字段值

**解决**: Ant Design Form 默认会保持字段值，除非调用 `resetFields()`

### 场景 3: 异步验证问题

**症状**: 快速点击"下一步"时，验证未完成

**检查**: `handleNext` 函数是否等待验证完成

```typescript
const handleNext = async () => {
  try {
    const currentFields = getCurrentStepFields();
    await form.validateFields(currentFields); // 等待验证完成
    setCurrentStep(currentStep + 1);
  } catch (error) {
    // Validation failed, stay on current step
  }
};
```

**状态**: ✅ 已正确实现异步验证
