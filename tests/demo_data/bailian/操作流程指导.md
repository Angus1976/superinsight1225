# 百联集团会员数据治理测试 — 操作流程指导

## 场景概述

客户：上海百联集团（百货、超市、购物中心三大板块）
目标：展示 SuperInsight 平台将原始异构会员数据转化为 AI 友好型数据后，通过 OpenClaw 技能在前端交互中获得质的飞跃。

---

## 第一步：数据导入（数据同步模块）

### 1.1 创建数据源

进入 **数据同步** → **数据源管理**（`/data-sync/sources`），添加 3 个数据源：

| 数据源名称 | 类型 | 对应文件 |
|-----------|------|---------|
| 百联百货会员系统 | `local_file` / `rest_api` | `raw/百货_会员表.json` + `raw/百货_消费流水.json` |
| 百联超市会员系统 | `local_file` / `rest_api` | `raw/超市_会员表.json` + `raw/超市_消费流水.json` |
| 百联购物中心系统 | `local_file` / `rest_api` | `raw/购物中心_会员表.json` + `raw/积分_兑换记录.json` |

### 1.2 执行同步

- 同步方向：`pull`
- 同步频率：`manual`（演示用手动触发）
- 数据进入 **临时数据**（`TempData`），状态为 `raw`

### 1.3 观察原始数据问题

在临时数据页面（`/data-lifecycle/temp-data`）可以看到：
- 性别编码混乱：`女` / `F` / `female` / `2` / `1` / `M` / `male`
- 手机号格式不一：`13812345678` / `139-2345-6789` / `158 0123 4567` / `+86-138-1234-5678`
- 日期格式不一：`1985-06-15` / `1978/11/03` / `19850615` / `Jun 15, 1985`
- 会员等级体系不同：`金卡/银卡/钻石卡/普卡` vs `GOLD/SILVER/NORMAL` vs `DIAMOND/PLATINUM/GOLD/SILVER/BASIC`
- 同一会员在 3 个系统中有不同 ID，无法直接关联

## 第二步：数据结构化与审核（数据流转模块）

### 2.1 结构化处理

在 **数据流转** → **临时数据** 中：
1. 选择导入的原始数据
2. 执行结构化操作，状态变为 `structured`
3. 平台自动识别字段类型、统一编码

### 2.2 审核入库

1. 业务专家审核结构化后的数据
2. 审核通过后，状态变为 `approved` → `in_sample_library`
3. 数据进入 **样本库**（`/data-lifecycle/samples`）

---

## 第三步：AI 预标注 + 人工标注（标注任务模块）

### 3.1 创建标注任务

进入 **标注任务**（`/tasks`）→ 新建任务：

| 任务名称 | 标注类型 | 数据源 | 标注内容 |
|---------|---------|--------|---------|
| 会员身份去重合并 | `entity_recognition` | 样本库 | 跨系统同一会员识别（基于手机号匹配） |
| 消费偏好标注 | `classification` | 样本库 | 消费品类偏好、品牌忠诚度、价格敏感度 |
| 会员生命周期标注 | `classification` | 样本库 | 新客/成长/活跃/沉默/预流失/已流失 |

### 3.2 AI 预标注

在 **AI 辅助标注面板**（`AIAnnotationPanel`）中：
1. 选择 LLM 模型进行预标注
2. AI 自动识别跨系统同一会员（基于手机号 + 姓名模糊匹配）
3. AI 预判消费偏好和生命周期阶段

### 3.3 业务专家审核

1. 业务专家在标注界面（`/tasks/:id/annotate`）审核 AI 预标注结果
2. 修正错误标注（如 AI 可能误判某些边界情况）
3. 审核通过后，标注数据状态变为 `annotated`

---

## 第四步：数据增强 — 业务规则注入（数据增强模块）

### 4.1 配置增强规则

进入 **数据增强**（`/augmentation/config`），配置以下规则：

| 规则名称 | 类型 | 说明 |
|---------|------|------|
| RFM 模型计算 | `feature_extraction` | 计算每个会员的 R/F/M 评分和价值分层 |
| 生命周期规则 | `normalization` | 基于消费行为判定生命周期阶段 |
| 跨板块关联 | `data_augmentation` | 计算跨板块消费指数和迁移路径 |
| 流失预警模型 | `feature_extraction` | 5 维流失信号综合评分 |
| 营销响应预测 | `data_augmentation` | 基于历史活动参与计算响应概率 |

规则定义参见：`enriched/business_rules.json`

### 4.2 执行增强

1. 在 **AI 处理**（`/augmentation/ai-processing`）页面执行
2. 增强后数据状态变为 `enhanced`
3. 输出结果对应：
   - `enriched/unified_member.json` — 统一会员实体
   - `enriched/member_profile.json` — 会员画像
   - `enriched/consumption_behavior.json` — 标准化消费行为

### 4.3 知识图谱构建

增强数据同步写入 Neo4j 知识图谱：
- 会员节点 + 品类节点 + 门店节点 + 品牌节点
- PREFERS / FREQUENTS / LOYAL_TO / CROSS_PURCHASE 关系
- 参见：`enriched/knowledge_graph.json`

---

## 第五步：AI 试验验证（数据流转模块）

### 5.1 创建 AI 试验

在 **数据流转** → **AI 试验**（`/data-lifecycle/trials`）中：
1. 选择增强后的数据集
2. 配置试验参数（LLM 模型、评估指标）
3. 运行试验，验证数据质量

### 5.2 质量评估

在 **质量管理**（`/quality/reports`）中查看：
- 数据完整性评分
- 标注一致性评分
- Ragas 语义质量评估

---

## 第六步：配置 OpenClaw 技能（智能服务引擎）

### 6.1 注册技能

通过管理后台（`/admin/ai-integration`）注册以下 OpenClaw 技能：

| 技能 ID | 名称 | 用途 |
|---------|------|------|
| `member-360-profile` | 会员360画像 | 查询单个会员完整画像 |
| `precision-marketing` | 精准营销推荐 | 生成个性化营销方案 |
| `churn-early-warning` | 流失预警 | 识别流失风险会员 |
| `cross-segment-insight` | 跨板块洞察 | 分析跨业态消费行为 |
| `member-segment-query` | 会员圈选 | 多维标签组合筛选 |

### 6.2 配置工作流

在 **AI 智能助手** → **工作流管理** 中创建工作流：

```json
{
  "name": "百联会员智能分析",
  "skill_ids": [
    "member-360-profile",
    "precision-marketing",
    "churn-early-warning",
    "cross-segment-insight",
    "member-segment-query"
  ],
  "data_source_auth": [
    {"source_id": "tasks", "tables": ["*"]},
    {"source_id": "data_lifecycle", "tables": ["*"]},
    {"source_id": "augmentation", "tables": ["*"]}
  ],
  "output_modes": ["merge"],
  "visible_roles": ["admin", "business_expert"]
}
```

## 第七步：前端交互演示（AI 智能助手）

### 7.1 进入 AI 智能助手

打开 `/ai-assistant` 页面，选择工作流"百联会员智能分析"。

### 7.2 演示对比（3 个场景）

依次输入以下问题，展示 AI 友好型数据带来的效果提升：

**场景 1 — 会员个体分析**
```
帮我分析会员王丽的消费情况，给出营销建议
```
对比参见：`demo_conversations/comparison_1_member_analysis.json`

**场景 2 — 营销活动策划**
```
母亲节快到了，帮我策划一个针对高价值女性会员的营销活动
```
对比参见：`demo_conversations/comparison_2_marketing_campaign.json`

**场景 3 — 流失预警**
```
最近3个月有哪些高价值会员有流失风险？
```
对比参见：`demo_conversations/comparison_3_churn_warning.json`

### 7.3 效果对比要点

| 维度 | 原始数据 | AI友好型数据 |
|------|---------|-------------|
| 数据关联 | 各系统孤立，无法跨板块 | 统一ID，跨3板块完整关联 |
| 会员识别 | 性别/日期编码混乱 | 标准化，含年龄段/会员时长 |
| 价值判断 | 无统一标准 | RFM量化评分 + 价值分层 |
| 生命周期 | 无 | 6阶段精准判定 |
| 消费洞察 | 零散流水 | 品类分布/品牌忠诚/消费趋势 |
| 流失预警 | 无法识别 | 5维信号量化 + 优先级排序 |
| 营销建议 | 泛泛而谈 | 精准到人/渠道/时机/预算/ROI |
| 跨板块联动 | 不可能 | 自动识别迁移路径和联动机会 |

## 演示话术建议

开场：
> "百联集团的会员数据分散在百货、超市、购物中心三大系统中，格式不统一、无法关联。我们通过 SuperInsight 平台的数据治理能力，将这些原始数据转化为 AI 友好型的结构化数据，让 AI 能够真正理解业务、给出可执行的建议。"

对比演示时：
> "同样一个问题，我们先看用原始数据 AI 能给出什么回答……现在我们看经过 SuperInsight 治理后的数据，AI 能给出什么回答……"

收尾：
> "数据质量决定了 AI 的上限。SuperInsight 不只是标注工具，而是将业务知识（RFM模型、生命周期规则、跨板块关联）系统化地注入数据，让 AI 从'能说话'变成'说对话'。"
