# SuperInsight 数据全流程架构说明

**最后更新**: 2026-03-12  
**维护者**: 开发团队  
**状态**: 已实现（约 80%）

---

## 概述

SuperInsight 平台实现了从数据获取到最终使用的完整数据生命周期管理。数据在各模块间基于 RBAC 权限体系流转，经历 **获取 → 临时存储 → 审核 → 样本库 → 标注 → 增强 → AI试验** 的全流程。

---

## 模块详解

### 1. 数据获取（Data Sync）

从外部数据源获取原始数据，是整个流程的入口。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-sync/sources` |
| 后端 API | `GET/POST /api/v1/data-sync/sources` |
| 数据源类型 | 数据库(database)、API(api)、文件(file)、数据流(stream) |
| 扩展类型 | 数据流转模块（TempData/Sample/Enhancement/AnnotationTask/AITrial） |
| 关键操作 | 创建数据源、手动/定时同步、启用/禁用、安全配置 |
| 安全特性 | AES-256-GCM 加密、JWT 认证、PII 检测、审计日志 |

### 2. 临时数据存储（Temp Data）

同步后的数据首先进入临时数据表，等待审核。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-lifecycle/temp-data` |
| 后端 API | `/api/v1/data-lifecycle/temp-data` |
| 数据模型 | `TempDataModel`（SQLAlchemy）→ `TempData`（Pydantic） |
| 状态流转 | `raw` → `structured` → `temp_stored` → `under_review` |
| 关键字段 | data_type, content(JSON), source, state, metadata |

### 3. 审核工作流（Review）

临时数据经审核后决定是否进入样本库。

| 项目 | 说明 |
|------|------|
| 后端 API | `/api/v1/data-lifecycle/reviews` |
| 审核状态 | `pending` → `in_progress` → `approved` / `rejected` |
| 权限要求 | `review.approve`、`review.reject`、`review.cancel` |
| 审核通过 | 数据状态变为 `approved`，可转入样本库 |

### 4. 样本库（Sample Library）

审核通过的数据进入样本库，带质量评分，是后续标注和增强的基础。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-lifecycle/samples` |
| 后端 API | `/api/v1/data-lifecycle/samples` |
| 数据模型 | `SampleModel` → `Sample` |
| 状态 | `in_sample_library` |
| 关键字段 | quality_score(0-100), annotation_count, version, tags(JSON) |

### 5. 标注任务（Annotation Tasks）

从样本创建标注任务，集成 Label Studio 进行专业标注。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-lifecycle/tasks`（数据流转内）、`/tasks`（独立标注管理） |
| 后端 API | `/api/v1/data-lifecycle/annotation-tasks`、`/api/v1/tasks` |
| 数据模型 | `AnnotationTaskModel` → `AnnotationTask` |
| 标注类型 | 分类、实体识别、关系抽取、情感分析、自定义 |
| 状态流转 | `created` → `in_progress` → `completed` / `cancelled` |
| 数据源选择 | 支持从数据同步、数据流转、数据增强三大模块选择数据源 |

### 6. 数据增强（Enhancement / Augmentation）

对标注后的数据进行增强处理，提升数据质量和数量。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-lifecycle/enhancement`（流转内）、`/augmentation`（独立模块） |
| 后端 API | `/api/v1/data-lifecycle/enhancement`、`/api/v1/augmentation/jobs` |
| 增强类型 | 数据扩充、质量改进、降噪、特征提取、归一化 |
| 作业状态 | `queued` → `running` → `completed` / `failed` / `cancelled` |
| 关键指标 | source_count, output_count, progress, quality_score |

### 7. AI 试验（AI Trial）

在增强后的数据上运行 AI 实验，评估模型效果。

| 项目 | 说明 |
|------|------|
| 前端路由 | `/data-lifecycle/trials` |
| 后端 API | `/api/v1/data-lifecycle/ai-trials` |
| 数据模型 | `AITrial`（Pydantic） |
| 数据阶段 | temp_table, sample_library, data_source, annotated, enhanced |
| 试验状态 | `created` → `running` → `completed` / `failed` |
| 关键指标 | model_name, metrics(JSON), data_stage |

## 数据状态枚举（DataState）

完整的 14 个状态，覆盖数据从原始到归档的全生命周期：

```
raw → structured → temp_stored → under_review → approved/rejected
  → in_sample_library → annotation_pending → annotating → annotated
  → enhancing → enhanced → trial_calculation → archived
```

## 权限控制体系

### 三层权限架构

1. **系统级权限**（`Permission` 枚举）：标注、任务、项目、用户、租户、工作空间等
2. **数据流转权限**（`DATA_LIFECYCLE_ROLE_PERMISSIONS`）：`module.action` 格式
3. **租户/工作空间隔离**：确保数据在租户和工作空间间隔离

### 四个角色权限矩阵

| 操作 | ADMIN | BUSINESS_EXPERT | ANNOTATOR | VIEWER |
|------|-------|-----------------|-----------|--------|
| 数据创建/编辑 | ✅ | ✅ | ✅ | ❌ |
| 数据删除 | ✅ | ❌ | ❌ | ❌ |
| 归档/恢复 | ✅ | ✅ | ❌ | ❌ |
| 审核（批准/拒绝） | ✅ | ✅ | ❌ | ❌ |
| 增强操作 | ✅ | ✅ | ❌ | ❌ |
| 增强回滚 | ✅ | ❌ | ❌ | ❌ |
| AI试验（创建/运行） | ✅ | ✅ | ❌ | ❌ |
| AI试验（查看） | ✅ | ✅ | ✅ | ✅ |
| 标注任务操作 | ✅ | ✅ | 部分 | ❌ |

### 权限检查机制

```typescript
// authStore.hasPermission 方法
// 带 "." 的权限 → 走 DATA_LIFECYCLE_ROLE_PERMISSIONS 映射
// 不带 "." 的权限 → 走系统级 Permission 枚举
hasPermission('review.approve')      // → 数据流转权限
hasPermission('view_annotation')     // → 系统级权限
```

## 横切关注点

### 版本控制（Version）
- 每次数据变更自动创建版本记录
- 变更类型：initial, annotation, enhancement, correction, merge
- 支持版本回溯和对比

### 审计日志（Audit Log）
- 记录所有操作：create, read, update, delete, transfer, state_change
- 操作结果：success, failure, partial
- 包含操作人、时间、IP、详情等完整审计信息
- 独立的转存审计日志（TransferAuditLog）
