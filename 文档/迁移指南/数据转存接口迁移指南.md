# 数据转存接口迁移指南

**版本**: 1.0  
**更新日期**: 2026-03-10  
**废弃截止**: 2026-06-10

---

## 概述

旧接口 `POST /api/enhancements/{job_id}/add-to-library` 已废弃，请迁移到统一接口 `POST /api/data-lifecycle/transfer`。

**时间线**：
- 2026-03-10：新接口上线，旧接口标记废弃
- 2026-06-10：旧接口下线，返回 `410 Gone`

旧接口当前仍可用，但响应会包含以下废弃警告头：

```
X-Deprecated-Endpoint: true
X-New-Endpoint: /api/data-lifecycle/transfer
X-Deprecation-Date: 2026-06-10
```

---

## 请求对比

### 旧接口

```http
POST /api/enhancements/{job_id}/add-to-library
Content-Type: application/json

{
  "user_id": "user-123"
}
```

### 新接口

```http
POST /api/data-lifecycle/transfer
Content-Type: application/json
Authorization: Bearer <token>
Accept-Language: zh-CN

{
  "source_type": "augmentation",
  "source_id": "<job_id>",
  "target_state": "in_sample_library",
  "data_attributes": {
    "category": "enhanced_data",
    "tags": ["augmentation"],
    "quality_score": 0.8
  },
  "records": [
    {
      "id": "record-1",
      "content": { ... },
      "metadata": { ... }
    }
  ]
}
```

**关键变化**：
- `job_id` 从 URL 路径移到请求体 `source_id` 字段
- `user_id` 改为 `Authorization` Bearer Token 认证
- 需要显式指定 `source_type`、`target_state` 和 `records`
- 支持 `Accept-Language` 头控制响应语言

## 响应对比

### 旧接口响应 (201)

```json
{
  "id": "lifecycle-uuid",
  "data_id": "enhanced-data-id",
  "content": { ... },
  "category": "enhanced_data",
  "quality_overall": 0.85
}
```

### 新接口响应 (200)

```json
{
  "success": true,
  "transferred_count": 1,
  "lifecycle_ids": ["lifecycle-uuid"],
  "target_state": "in_sample_library",
  "message": "已成功将 1 条记录转存到样本库",
  "navigation_url": "/data-lifecycle/temp-data"
}
```

需要审批时返回 (202)：

```json
{
  "success": true,
  "approval_required": true,
  "approval_id": "approval-uuid",
  "message": "转存请求已提交，等待审批"
}
```

## 迁移示例（Python）

### 迁移前

```python
import requests

resp = requests.post(
    f"{BASE_URL}/api/enhancements/{job_id}/add-to-library",
    json={"user_id": "user-123"}
)
library_id = resp.json()["id"]
```

### 迁移后

```python
import requests

resp = requests.post(
    f"{BASE_URL}/api/data-lifecycle/transfer",
    headers={
        "Authorization": f"Bearer {token}",
        "Accept-Language": "zh-CN"
    },
    json={
        "source_type": "augmentation",
        "source_id": job_id,
        "target_state": "in_sample_library",
        "data_attributes": {
            "category": "enhanced_data",
            "tags": ["augmentation"],
            "quality_score": 0.8
        },
        "records": records  # 需要自行构造记录列表
    }
)
result = resp.json()

if result.get("approval_required"):
    print(f"需要审批，审批ID: {result['approval_id']}")
else:
    lifecycle_ids = result["lifecycle_ids"]
```

## 新接口额外能力

| 能力 | 旧接口 | 新接口 |
|------|--------|--------|
| 目标状态选择 | 仅样本库 | 临时存储 / 样本库 / 待标注 |
| 数据源类型 | 仅增强 | 结构化 / 增强 / 同步 / 标注 / AI助手 |
| 权限控制 | 无 | 基于角色的权限矩阵 |
| 审批流程 | 无 | 自动触发审批 |
| 批量转存 | 不支持 | `POST /api/data-lifecycle/batch-transfer` |
| 国际化 | 不支持 | `Accept-Language` 头 |
| 审计日志 | 无 | 自动记录 |

## 常见问题

**Q: 旧接口什么时候完全下线？**  
A: 2026-06-10 之后旧接口将返回 `410 Gone`。

**Q: 新接口需要哪些额外参数？**  
A: 主要是 `source_type`、`target_state`、`data_attributes` 和 `records`。旧接口只需 `user_id`，新接口需要完整的转存信息。

**Q: 如何判断是否需要审批？**  
A: 响应状态码 `202` 且 `approval_required: true` 表示需要审批。可提前调用 `GET /api/data-lifecycle/permissions/check?operation=transfer_to_library` 检查权限。

**Q: 旧接口现在内部是怎么工作的？**  
A: 旧接口已内部转换为调用新的 `DataTransferService`，行为一致，但缺少权限控制和审批流程。
