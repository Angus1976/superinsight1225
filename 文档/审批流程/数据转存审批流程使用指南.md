# 数据转存审批流程使用指南

**版本**: 1.0  
**更新日期**: 2026-03-10

---

## 审批流程概览

```
用户提交转存请求
       │
       ▼
  ┌──────────┐    否     ┌──────────┐
  │ 需要审批？├─────────→│ 直接执行  │
  └────┬─────┘          └──────────┘
       │ 是
       ▼
  ┌──────────────┐
  │ 创建审批工单  │
  │ (PENDING)    │
  └──────┬───────┘
         │
    ┌────┴────┐
    ▼         ▼
 站内消息   邮件通知
    └────┬────┘
         │
         ▼
  ┌──────────────┐
  │ 审批人审核    │
  └──┬───────┬───┘
     │       │
     ▼       ▼
  批准     拒绝          7天无操作
     │       │               │
     ▼       ▼               ▼
  自动执行  通知用户      自动过期
  转存操作  拒绝原因     (EXPIRED)
```

## 触发审批的条件

| 条件 | 说明 |
|------|------|
| DATA_ANALYST 转存到样本库/待标注 | 数据分析师无直接权限 |
| USER 转存到样本库/待标注 | 普通用户无直接权限 |
| USER 创建数据记录 | 普通用户创建需审批 |
| DATA_ANALYST 更新数据记录 | 数据分析师更新需审批 |
| DATA_MANAGER 删除数据记录 | 数据管理员删除需审批 |
| 批量转存 > 1000 条记录 | 大批量操作安全控制 |

ADMIN 和 DATA_MANAGER 的转存操作无需审批。

---

## 审批状态说明

| 状态 | 英文 | 说明 |
|------|------|------|
| 待审批 | PENDING | 等待审批人处理 |
| 已批准 | APPROVED | 审批通过，转存自动执行 |
| 已拒绝 | REJECTED | 审批拒绝，用户收到拒绝原因 |
| 已过期 | EXPIRED | 超过 7 天未处理，自动拒绝 |

## 用户操作指南

### 提交转存请求

正常调用转存接口即可。系统会自动判断是否需要审批：

```bash
curl -X POST /api/data-lifecycle/transfer \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -H "Accept-Language: zh-CN" \
  -d '{
    "source_type": "augmentation",
    "source_id": "job-123",
    "target_state": "in_sample_library",
    "data_attributes": {
      "category": "商品信息",
      "tags": ["增强数据"],
      "quality_score": 0.9
    },
    "records": [
      {"id": "r1", "content": {"text": "示例数据"}}
    ]
  }'
```

需要审批时返回 `202`：

```json
{
  "success": true,
  "approval_required": true,
  "approval_id": "approval-uuid",
  "message": "转存请求已提交，等待审批",
  "estimated_approval_time": "2-3 个工作日"
}
```

### 查看我的审批状态

```bash
# 查看所有审批记录
curl -X GET "/api/data-lifecycle/approvals" \
  -H "Authorization: Bearer <token>"

# 按状态筛选
curl -X GET "/api/data-lifecycle/approvals?status=pending" \
  -H "Authorization: Bearer <token>"
```

响应示例：

```json
{
  "success": true,
  "approvals": [
    {
      "id": "approval-uuid",
      "status": "pending",
      "created_at": "2026-03-10T10:00:00",
      "expires_at": "2026-03-17T10:00:00",
      "transfer_request": {
        "source_type": "augmentation",
        "target_state": "in_sample_library",
        "record_count": 15
      }
    }
  ],
  "total": 1,
  "has_more": false
}
```

普通用户只能查看自己的审批记录。

## 审批人操作指南

审批人角色：ADMIN、DATA_MANAGER。

### 查看待审批列表

```bash
curl -X GET "/api/data-lifecycle/approvals?status=pending" \
  -H "Authorization: Bearer <token>"
```

管理员和数据管理员可查看所有用户的审批请求，也可按 `user_id` 筛选特定用户。

### 批准请求

```bash
curl -X POST "/api/data-lifecycle/approvals/{approval_id}/approve" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "approved": true,
    "comment": "数据质量符合要求，批准入库"
  }'
```

批准后系统自动执行转存操作，申请人收到通知。

### 拒绝请求

```bash
curl -X POST "/api/data-lifecycle/approvals/{approval_id}/approve" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "approved": false,
    "comment": "数据质量评分不足，请重新处理后再提交"
  }'
```

拒绝后申请人收到通知及拒绝原因。

## 通知机制

审批相关通知通过两个渠道发送：

- **站内消息**：实时推送到审批人/申请人的消息中心
- **邮件通知**：发送到审批人/申请人的注册邮箱

通知触发时机：
1. 创建审批请求 → 通知所有审批人
2. 批准/拒绝 → 通知申请人

---

## 超时机制

- 审批请求默认有效期 **7 天**
- 超时未处理的请求自动标记为 `EXPIRED`
- 系统通过定时任务检查并过期超时请求
- 过期后申请人需重新提交转存请求

## 常见问题

**Q: 审批被拒绝后怎么办？**  
查看拒绝原因，修正问题后重新提交转存请求。

**Q: 审批过期了怎么办？**  
重新提交转存请求即可，系统会创建新的审批工单。

**Q: 如何知道我的操作是否需要审批？**  
调用权限检查接口：

```bash
curl -X GET "/api/data-lifecycle/permissions/check?operation=transfer_to_library" \
  -H "Authorization: Bearer <token>"
```

返回 `requires_approval: true` 表示需要审批。

**Q: ADMIN 的操作也需要审批吗？**  
不需要。ADMIN 拥有所有权限，所有操作直接执行。
