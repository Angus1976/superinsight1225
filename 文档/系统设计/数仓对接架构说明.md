# 客户数仓对接架构说明

**最后更新**: 2026-03-14  
**维护者**: 开发团队  
**状态**: 架构设计

---

## 概述

本文档说明客户数据仓库（数仓）与 SuperInsight 平台的对接架构。

核心设计理念：**旁路模式，零干预生产系统**。

SuperInsight 以只读方式从客户数仓提取数据，在平台内独立完成标注、增强、AI 试验等处理，生成的 AI 友好型数据作为客户生产数据的"旁路副本"独立存在。客户通过 API 接口或文件导出获取处理结果，无需对现有数仓做任何改造。

```
+------------------------------------------+
|     Customer Data Warehouse (Production)  |
|          READ-ONLY, NO MODIFICATION       |
+--------------------+---------------------+
                     |
                     | SELECT (read-only)
                     v
+------------------------------------------+
|        SuperInsight Platform (Bypass)     |
|                                          |
|  Sync -> Lifecycle -> Annotation -> Aug  |
|                                   |      |
|                              AI Trial    |
|                                   |      |
|          AI-Friendly Data (stored here)  |
|                   |                      |
|          +--------+--------+             |
|          v                 v             |
|     External API     File Export         |
|     (X-API-Key)     (Approval+WM)       |
+----------+-----------------+-------------+
           |                 |
           v                 v
    Client System      Data Team
    (real-time)       (offline)
```

---

## 为什么选择旁路模式

| 考量               | 旁路模式                   | 回写模式                         |
|:-------------------|:--------------------------|:---------------------------------|
| 对客户系统的影响     | 零影响，只读访问            | 需要写入权限，可能影响生产         |
| 客户 DBA 审批难度   | 低（只需 SELECT 权限）      | 高（需要 INSERT/UPDATE 权限）     |
| 数据安全风险        | 低（不触碰生产数据）         | 中（写入操作有风险）              |
| 部署复杂度          | 低（配置连接即可）           | 高（需要建表、映射、回写策略）     |
| 适用阶段            | 初期对接，快速验证           | 深度集成阶段（后续可扩展）        |

> 后续如客户需要将结果回写数仓，平台已预留输出同步能力（`OutputSyncService`），可按需开启。

## (1) 数据接入：只读提取客户数仓

### 支持的数仓/数据库类型

| 类别           | 支持的系统                              | 连接器                            |
|:--------------|:---------------------------------------|:---------------------------------|
| OLAP 数仓     | ClickHouse, Apache Doris (StarRocks)   | `connectors/datalake/`           |
| 数据湖        | Hive, Iceberg, Delta Lake              | `connectors/datalake/`           |
| 查询引擎      | Presto/Trino, SparkSQL                 | `connectors/datalake/`           |
| 关系型数据库   | MySQL, PostgreSQL, Oracle, SQL Server  | `connectors/database/`           |
| 文件/对象存储  | S3, FTP/SFTP, Local File               | `connectors/file/`               |
| API           | REST, GraphQL, SOAP, Webhook           | `connectors/api/`                |

> 连接器路径前缀: `src/sync/`

### 接入特性

| 特性         | 说明                           | 对应代码                                     |
|:------------|:-------------------------------|:--------------------------------------------|
| 只读安全     | 仅需 SELECT 权限，不修改客户数据 | `connectors/`                               |
| 增量同步     | 基于时间戳或 CDC 的增量提取      | `connectors/incremental_sync.py`            |
| 定时调度     | Cron 表达式配置自动同步频率      | `scheduler/`                                |
| 自动脱敏     | 同步过程中识别和脱敏敏感字段      | `desensitization/`                          |
| 断点续传     | 同步中断后从断点恢复             | `connectors/recovery_system.py`             |
| 连接健康监控  | 自动检测连接状态                | `connectors/database/health_monitor.py`     |
| 数据质量评分  | 同步数据自动质量评估             | `quality/`                                  |

> 代码路径前缀: `src/sync/`

### 配置流程

通过平台「数据同步 → 数据源管理」页面操作：

```
Step 1  选择数据源类型（如 ClickHouse）
Step 2  填写连接信息（主机、端口、数据库、认证凭据）
Step 3  测试连接
Step 4  配置同步策略（全量 / 增量）和调度计划
Step 5  选择需要同步的表 / 视图
```

## (2) 平台内处理：生成 AI 友好型数据

数据进入平台后，在独立环境中经过完整的 AI 数据治理流程。所有处理结果存储在 SuperInsight 平台内部，不影响客户生产数仓。

```
Synced Data --> Temp Storage --> Review --> Sample Library
                                               |
                          +--------------------+--------------------+
                          v                    v                    v
                    AI Pre-Label         Augmentation          AI Trial
                    (LLM Models)        (Expand/Denoise)     (Evaluation)
                          |                    |                    |
                          +--------+-----------+                    |
                                   v                                |
                          Human Review & Label                      |
                          (Label Studio)                            |
                                   |                                |
                                   +--------------------------------+
                                                |
                                                v
                                   AI-Friendly Data (Final)
                                   +-- Annotation Results
                                   +-- Augmented Data
                                   +-- Quality Reports
                                   +-- AI Trial Results
```

| 处理阶段   | 说明                                                       |
|:----------|:----------------------------------------------------------|
| 数据流转   | 临时存储 → 审核 → 入库，全程版本控制和审计日志                  |
| AI 预标注  | 集成 LLM 模型（Ollama/OpenAI）智能预标注，降低人工成本         |
| 人工标注   | 基于 Label Studio 的人机协作，支持文本分类、实体识别、关系抽取等 |
| 质量评估   | 基于 Ragas 框架的语义质量评估，自动评分                        |
| 数据增强   | 数据扩充、降噪、特征提取、归一化                               |
| AI 试验   | 模型效果验证，结果可反馈回标注/增强环节迭代优化                   |

## (3) 结果获取：两种输出路径

### (3-A) 外部 API — 推荐，实时访问

客户业务系统通过 RESTful API 实时查询处理结果，适合程序化集成场景。

#### API 端点

| 端点                                      | 数据类型     | 说明                       |
|:-----------------------------------------|:------------|:--------------------------|
| `GET /api/v1/external/annotations`       | 标注结果     | 文本分类、实体识别等标注数据 |
| `GET /api/v1/external/augmented-data`    | 增强数据     | 经过扩充、降噪处理的数据     |
| `GET /api/v1/external/quality-reports`   | 质量报告     | 数据质量评分和分析           |
| `GET /api/v1/external/experiments`       | AI 试验结果  | 模型评估和效果验证数据       |

#### 认证与安全

| 项目           | 说明                                                          |
|:--------------|:-------------------------------------------------------------|
| 认证方式       | 请求头 `X-API-Key`，密钥格式 `sk_` 前缀                        |
| 密钥安全       | SHA-256 哈希存储，完整密钥仅创建时可见一次                        |
| 权限范围       | 通过 `scopes` 控制可访问的端点（如仅允许访问标注结果）             |
| 速率限制       | 按密钥独立配置每分钟/每日配额，超限返回 `429` + `Retry-After`     |
| 密钥生命周期   | `active` ↔ `disabled` 可切换，`revoked` 为终态不可恢复          |
| 调用审计       | 每次调用记录端点、状态码、响应时间                                |

#### 响应格式

统一 JSON 分页响应，支持 `page`、`page_size`、`sort_by`、`fields`（字段筛选）参数：

```json
{
    "total": 1250,
    "page": 1,
    "page_size": 50,
    "items": [...]
}
```

#### 对应代码结构

| 组件       | 路径                                | 职责                            |
|:----------|:-----------------------------------|:-------------------------------|
| API 路由   | `gateway/external_data_router.py`  | 4 个数据端点 + 分页/排序/筛选    |
| 认证中间件  | `gateway/api_key_auth_middleware.py`| X-API-Key 验证                  |
| 密钥管理   | `gateway/api_key_service.py`       | 密钥 CRUD + 状态机              |
| 速率限制   | `gateway/rate_limiter.py`          | 按密钥配额控制                   |
| 数据模型   | `models.py` → `APIKeyModel`       | 密钥存储（哈希 + 前缀）          |
| 调用日志   | `models.py` → `APICallLogModel`   | 调用审计记录                     |

> 代码路径前缀: `src/sync/`

### (3-B) 文件导出 — 批量交付

通过导出控制模块将数据导出为文件，适合一次性交付或离线分析场景。

| 特性       | 说明                         | 对应代码                                |
|:----------|:----------------------------|:---------------------------------------|
| 审批流程   | 敏感数据导出需经审批           | `export_control/approval_workflow.py`  |
| 数字水印   | 导出文件自动添加水印，可追溯    | `export_control/watermark_service.py`  |
| 多种格式   | CSV, JSON, Excel 等          | `export_control/export_api.py`         |
| 权限控制   | 基于角色的导出权限             | `export_control/export_permission_service.py` |
| 导出监控   | 完整的导出审计日志             | `export_control/export_monitor.py`     |

> 代码路径前缀: `src/sync/`

## 安全保障

| 层级     | 措施                                                    |
|:--------|:-------------------------------------------------------|
| 接入层   | 只读访问客户数仓，仅需 SELECT 权限                        |
| 传输层   | HTTPS 加密传输                                          |
| 认证层   | API Key 认证（外部 API）/ 平台账号认证（导出）             |
| 权限层   | RBAC 角色权限 + scopes 数据范围控制 + 租户隔离             |
| 数据层   | 自动脱敏 + 数字水印 + 版本控制                            |
| 审计层   | 全操作审计日志 + API 调用日志 + 导出监控                   |

## 典型对接场景

### 场景一：数仓数据标注 + API 查询

```
Customer ClickHouse
        |
        | (1) Scheduled incremental sync (daily, read-only)
        v
SuperInsight (bypass processing)
        |
        | (2) AI pre-label + human review
        v
Client System
        |
        | (3) GET /api/v1/external/annotations
        |     X-API-Key: sk_xxxx
        |
        +-- Receive JSON annotation results
```

### 场景二：增强数据批量导出

```
Data Team
        |
        | (1) Submit export request (select augmented dataset)
        v
SuperInsight Export Approval
        |
        | (2) Approved, generate watermarked file
        v
Download CSV/JSON
        |
        +-- Import into internal analysis system
```

## 后续扩展：回写数仓（按需开启）

当客户进入深度集成阶段，可开启输出同步能力，将处理结果直接写回客户数仓。

| 能力       | 说明                           | 对应代码                                  |
|:----------|:-------------------------------|:-----------------------------------------|
| 输出同步   | 全量/增量写回目标数仓            | `push/output_sync_service.py`            |
| 字段映射   | 平台字段 → 数仓字段的可视化映射  | `transformer/field_mapper.py`            |
| 写入验证   | 写入前自动校验数据类型和格式      | `OutputSyncService.validate_output_data()`|
| 断点续传   | 大批量写入中断后可恢复           | `OutputSyncService.resume_from_checkpoint()` |
| 失败告警   | 失败率超阈值自动告警             | `push/output_sync_alert_service.py`      |
| 结果校验   | 写入后自动验证数据一致性          | `push/result_verifier.py`                |

> 代码路径前缀: `src/sync/`
>
> 此能力已在代码层面实现，`SyncJobModel` 支持 `direction=push/bidirectional`，客户需要时可通过平台配置开启。
