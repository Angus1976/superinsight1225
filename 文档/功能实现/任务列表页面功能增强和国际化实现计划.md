## 核心功能说明

### 功能 1: 刷新功能增强 (P0)

**当前问题**: 刷新按钮只刷新本地任务列表，不同步 Label Studio 项目状态

**解决方案**:
1. 调用 Label Studio API 获取所有项目
2. 对比本地任务和 Label Studio 项目
3. 同步项目信息（名称、任务数量、标注进度）
4. 更新本地任务的同步状态和时间
5. 显示同步进度和结果

**实现文件**:
- `frontend/src/services/labelStudioService.ts` (新建)
- `frontend/src/pages/Tasks/index.tsx` (更新)

### 功能 2: 导出数据功能 (P1)

**需求**: 支持导出任务列表和标注结果

**支持格式**:
- CSV: 简单数据导出
- JSON: 完整数据导出（可选包含标注结果）
- Excel: 多 sheet 报表导出

**实现文件**:
- `frontend/src/services/exportService.ts` (新建)
- `frontend/src/pages/Tasks/index.tsx` (更新)

**依赖**: 需要安装 `xlsx` 包

### 功能 3: 国际化支持 (P0)

**需求**: 
- 前端默认中文界面
- 支持中英文切换
- Label Studio 跟随前端语言切换

**实现方案**:
1. 创建翻译文件 (`zh/tasks.json`, `en/tasks.json`)
2. 配置 i18n 默认语言为中文
3. 创建 `useLabelStudioUrl` hook，自动添加语言参数
4. 更新所有组件使用翻译键

**实现文件**:
- `frontend/src/locales/zh/tasks.json` (新建)
- `frontend/src/locales/en/tasks.json` (新建)
- `frontend/src/utils/labelStudioUrl.ts` (新建)
- `frontend/src/i18n/config.ts` (更新)
- `frontend/src/pages/Tasks/*.tsx` (更新)

## 技术要点

### 1. 遵循 i18n 翻译规范

**必须遵守**: `.kiro/steering/i18n-translation-rules.md`

**关键规则**:
- 区分对象和字符串类型翻译键
- 避免重复翻译键
- 使用 camelCase 命名
- 保持中英文文件结构一致

### 2. Label Studio 语言同步

**实现方式**: URL 参数传递

```typescript
// 前端语言 → Label Studio 语言
const languageMap = {
  'zh': 'zh-cn',  // 中文简体
  'en': 'en',     // 英文
};

// URL 示例
const url = `http://localhost:8080/projects/123/data?lang=zh-cn`;
```

### 3. 同步逻辑

**核心流程**:
```
1. 获取 Label Studio 项目列表
   ↓
2. 对比本地任务
   ↓
3. 同步项目信息
   ↓
4. 更新本地数据
   ↓
5. 刷新列表显示
```
