# 数据转存权限配置文档

**版本**: 1.0  
**更新日期**: 2026-03-10

---

## 角色说明

| 角色 | 代码 | 说明 |
|------|------|------|
| 管理员 | `admin` | 拥有所有权限，无需审批 |
| 数据管理员 | `data_manager` | 大部分操作无需审批，删除需审批 |
| 数据分析师 | `data_analyst` | 基础操作，部分需审批 |
| 普通用户 | `user` | 有限操作，大部分需审批 |

---

## 转存操作权限矩阵

控制数据在不同目标状态间的转存权限。

| 操作 | 管理员 | 数据管理员 | 数据分析师 | 普通用户 |
|------|--------|-----------|-----------|---------|
| 转存到临时存储 (`temp_stored`) | ✅ | ✅ | ✅ | ✅ |
| 转存到样本库 (`in_sample_library`) | ✅ | ✅ | ⚠️ 需审批 | ⚠️ 需审批 |
| 转存到待标注 (`annotation_pending`) | ✅ | ✅ | ⚠️ 需审批 | ⚠️ 需审批 |
| 批量转存 (`batch_transfer`) | ✅ | ✅ | ❌ | ❌ |
| 跨项目转存 (`cross_project`) | ✅ | ❌ | ❌ | ❌ |

**特殊规则**：
- 单次转存超过 **1000 条**记录时，自动触发 `batch_transfer` 权限检查
- 跨项目转存需要 `cross_project` 权限

## CRUD 操作权限矩阵

控制对流转数据的增删改查及合并拆分操作。

| 操作 | 管理员 | 数据管理员 | 数据分析师 | 普通用户 |
|------|--------|-----------|-----------|---------|
| 创建 (`create`) | ✅ | ✅ | ✅ | ⚠️ 需审批 |
| 查询 (`read`) | ✅ | ✅ | ✅ | ✅ |
| 更新 (`update`) | ✅ | ✅ | ⚠️ 需审批 | ❌ |
| 删除 (`delete`) | ✅ | ⚠️ 需审批 | ❌ | ❌ |
| 合并 (`merge`) | ✅ | ✅ | ❌ | ❌ |
| 拆分 (`split`) | ✅ | ✅ | ❌ | ❌ |

## 权限检查 API

### 接口信息

```
GET /api/data-lifecycle/permissions/check
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `source_type` | string | 是 | 数据源类型：`structuring`, `augmentation`, `sync`, `annotation`, `ai_assistant`, `manual` |
| `target_state` | string | 是 | 目标状态：`temp_stored`, `in_sample_library`, `annotation_pending` |
| `operation` | string | 否 | 操作类型：`transfer`（默认）, `batch_transfer` |

### 请求头

```
Authorization: Bearer <token>
Accept-Language: zh-CN | en-US
```

### 响应示例

**有权限，无需审批**：
```json
{
  "allowed": true,
  "requires_approval": false,
  "user_role": "admin"
}
```

**有权限，需审批**：
```json
{
  "allowed": true,
  "requires_approval": true,
  "user_role": "data_analyst",
  "reason": "转存请求已提交，等待审批"
}
```

**无权限**：
```json
{
  "allowed": false,
  "requires_approval": false,
  "user_role": "user",
  "reason": "您没有权限执行此操作"
}
```

## 使用示例

### 前端：操作前检查权限

```typescript
import { checkPermissionAPI } from '@/api/dataLifecycle';

const result = await checkPermissionAPI({
  source_type: 'structuring',
  target_state: 'in_sample_library',
  operation: 'transfer'
});

if (!result.allowed) {
  // 禁用按钮或提示无权限
} else if (result.requires_approval) {
  // 提示用户该操作需要审批
}
```

### 后端：权限检查

```python
from src.services.permission_service import PermissionService, UserRole

service = PermissionService()

# 检查转存权限
result = service.check_permission(
    user_role=UserRole.DATA_ANALYST,
    target_state="in_sample_library",
    record_count=500
)
# result.allowed=True, result.requires_approval=True

# 检查 CRUD 权限
result = service.check_crud_permission(
    user_role=UserRole.DATA_MANAGER,
    operation="delete"
)
# result.allowed=True, result.requires_approval=True
```

## 审批触发条件

以下情况会自动触发审批流程：

1. **数据分析师/普通用户**转存到样本库或待标注
2. **普通用户**创建数据记录
3. **数据分析师**更新数据记录
4. **数据管理员**删除数据记录
5. 单次转存超过 **1000 条**记录（批量转存权限检查）

审批超时时间默认 **7 天**，超时自动拒绝。

## 权限矩阵配置

权限矩阵定义在 `src/services/permission_service.py` 的 `PermissionService` 类中：

- `PERMISSION_MATRIX` — 转存操作权限
- `CRUD_PERMISSION_MATRIX` — CRUD 操作权限

如需调整权限配置，修改对应矩阵中的 `allowed` 和 `requires_approval` 字段即可。
