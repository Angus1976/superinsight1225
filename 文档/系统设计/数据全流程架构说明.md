# SuperInsight 数据全流程架构说明

**最后更新**: 2026-03-12  
**维护者**: 开发团队  
**状态**: 已实现（约 85%）

---

## 概述

SuperInsight 平台的数据流转采用**网状互联架构**，而非线性管道。各模块（数据同步、数据流转、数据增强、标注任务）之间基于 RBAC 权限体系灵活互通，数据可在任意模块间循环流转、反复优化。

核心设计理念：**任何模块的输出都可以作为其他模块的输入**。

```
|       |                     |                     |        |
|       |        +------------+---------------------+        |
|       |        v            v                              |
|       |  +----------+  +----------+                        |
|       +->|Enhance-  |<>| AI Trial |                        |
|          |ment      |  |          |                        |
|          +----------+  +----------+                        |
|                                                            |

+------------------------+-----------------------------------+

     | as source | | +External| | as source  |
     +-----------+ +----------+ +------------+

Cross-cutting: Version Control + Audit Log + RBAC
```

## 模块间互通关系

这是与线性流程最大的区别——各模块间存在双向或多向数据引用：

### 标注任务创建 → 可从任意模块选择数据源

`TaskCreateModal` 的数据源选择支持 3 大模块 7 种数据类型：

| 分组           | 数据源类型                        | 权限要求                |
|:--------------|:---------------------------------|:-----------------------|
| 基础类型       | 文件上传、API、数据库              | —                      |
| 数据同步模块   | `ds_sources` (同步数据源)         | `dataLifecycle.create` |
| 数据流转模块   | `lc_temp_data` (临时数据)         | `dataLifecycle.create` |
| 数据流转模块   | `lc_sample` (样本库)              | `dataLifecycle.create` |
| 数据流转模块   | `lc_annotation` (标注任务)        | `annotationTask.create`|
| 数据流转模块   | `lc_enhancement` (增强数据)       | `enhancement.create`   |
| 数据流转模块   | `lc_ai_trial` (AI 试验)          | `aiTrial.view`         |
| 数据增强模块   | `aug_samples` (增强样本)          | `enhancement.create`   |

### 数据同步 → 可添加数据流转模块为数据源类型

`DataSync/Sources` 的"添加数据源"支持 5 种数据流转子类型：

| 数据源类型                  | 权限要求                |
|:--------------------------|:-----------------------|
| `lifecycle_temp_data`     | `dataLifecycle.create` |
| `lifecycle_sample`        | `dataLifecycle.create` |
| `lifecycle_enhancement`   | `enhancement.create`   |
| `lifecycle_annotation`    | `annotationTask.create`|
| `lifecycle_ai_trial`      | `aiTrial.view`         |

### 数据流转模块内部 → 子模块间状态流转

```
TempData --Review--> Samples --Create Task--> Annotation
    |                    |                        |
    |                    v                        v
    +--------------> Enhancement <---------- Results
                         |
                         v
                     AI Trial --Feedback--> Back to Enhancement/Annotation
```

## 各模块说明

### 1. 数据同步 (Data Sync)

外部数据获取入口，同时可将数据流转模块的数据作为同步源。

| 项目         | 说明                                                                                    |
|:------------|:---------------------------------------------------------------------------------------|
| 路由         | `/data-sync/sources`, `/data-sync/history`, `/data-sync/scheduler`                     |
| API         | `GET/POST /api/v1/data-sync/sources`                                                   |
| 基础源类型   | database, api, file, stream                                                             |
| 扩展源类型   | lifecycle_temp_data, lifecycle_sample, lifecycle_enhancement, lifecycle_annotation, lifecycle_ai_trial |

### 2. 数据流转 (Data Lifecycle)

数据生命周期管理的核心模块，包含 5 个子模块和审计日志。

| 子模块   | 路由                            | API                                       | 状态流转                                        |
|:--------|:-------------------------------|:------------------------------------------|:-----------------------------------------------|
| 临时数据 | `/data-lifecycle/temp-data`    | `/api/v1/data-lifecycle/temp-data`        | raw → structured → temp_stored → under_review  |
| 样本库   | `/data-lifecycle/samples`      | `/api/v1/data-lifecycle/samples`          | in_sample_library (含 quality_score)            |
| 标注任务 | `/data-lifecycle/tasks`        | `/api/v1/data-lifecycle/annotation-tasks` | created → in_progress → completed/cancelled    |
| 数据增强 | `/data-lifecycle/enhancement`  | `/api/v1/data-lifecycle/enhancement`      | queued → running → completed/failed            |
| AI 试验 | `/data-lifecycle/trials`       | `/api/v1/data-lifecycle/ai-trials`        | created → running → completed/failed           |
| 审计日志 | `/data-lifecycle/audit`        | —                                         | 记录所有操作                                    |

### 3. 数据增强 (Augmentation)

独立的数据增强模块，与数据流转中的增强子模块互补。既可处理平台内已有数据（样本库、标注结果等），也可直接处理来自外部的数据（文件上传、API 导入）。

| 项目       | 说明                                                                                  |
|:----------|:-------------------------------------------------------------------------------------|
| 路由       | `/augmentation/samples`, `/augmentation/config`, `/augmentation/ai-processing`       |
| API       | `/api/v1/augmentation/jobs`, `/api/v1/augmentation/samples`, `/api/v1/augmentation/upload` |
| 增强类型   | 数据扩充、质量改进、降噪、特征提取、归一化                                                 |
| 数据来源   | 内部数据 (样本库/标注结果/流转模块) + 外部数据 (文件上传)                                   |

### 4. 标注任务管理 (Tasks)

独立的标注任务管理，集成 Label Studio，可从任意模块拉取数据。

| 项目       | 说明                                                    |
|:----------|:-------------------------------------------------------|
| 路由       | `/tasks`, `/tasks/:id`, `/tasks/:id/annotate`          |
| API       | `/api/v1/tasks`                                        |
| 标注类型   | 文本分类、实体识别、关系抽取、情感分析、自定义              |
| 数据源     | 支持从数据同步、数据流转 (5 种)、数据增强模块选择          |

## 数据状态枚举 (DataState)

14 个状态覆盖完整生命周期，但数据不必按顺序经历所有状态：

```
raw --> structured --> temp_stored --> under_review --> approved / rejected
                                                            |
                                                    in_sample_library
                                                            |
                                      annotation_pending --> annotating --> annotated
                                                                               |
                                                             enhancing --> enhanced
                                                                               |
                                                                   trial_calculation
                                                                               |
                                                                          archived
```

数据可在 `annotated <-> enhancing` 和 `enhanced <-> trial_calculation` 之间反复迭代。

## 权限控制体系

### 双轨权限机制

```typescript
// authStore.hasPermission(permission)
// With "."  --> DATA_LIFECYCLE_ROLE_PERMISSIONS (Data Lifecycle permissions)
// Without "." --> Permission enum (System-level permissions)
hasPermission('review.approve')    // Data Lifecycle
hasPermission('view_annotation')   // System-level
```

### 四角色权限矩阵（数据流转模块）

| 操作                | ADMIN | BUSINESS_EXPERT | ANNOTATOR | VIEWER |
|:-------------------|:-----:|:---------------:|:---------:|:------:|
| 数据创建/编辑       |  ✅   |       ✅        |    ✅     |   ❌   |
| 数据删除            |  ✅   |       ❌        |    ❌     |   ❌   |
| 归档/恢复           |  ✅   |       ✅        |    ❌     |   ❌   |
| 审核操作            |  ✅   |       ✅        |    ❌     |   ❌   |
| 增强操作            |  ✅   |       ✅        |    ❌     |   ❌   |
| 增强回滚            |  ✅   |       ❌        |    ❌     |   ❌   |
| AI 试验 (创建/运行) |  ✅   |       ✅        |    ❌     |   ❌   |
| AI 试验 (查看)      |  ✅   |       ✅        |    ✅     |   ✅   |
| 标注任务            |  ✅   |       ✅        |   部分    |   ❌   |

### 三层隔离

1. 系统级 RBAC（UserRole + Permission 枚举）
2. 数据流转权限（`module.action` 格式，动态过滤可用数据源）
3. 租户/工作空间隔离（确保多租户数据安全）

## AI 智能助手数据源集成

### 架构概述

AI 智能助手通过权限控制的数据源访问机制，结合 LLM（Ollama/OpenAI）和 OpenClaw 技能引擎，实现基于平台治理数据的智能分析。

```
User Chat --> AI Assistant Page (select data sources + output mode)
                  |
                  v
          Backend ChatRequest (data_source_ids + output_mode)
                  |
                  +-- build_data_context() --> Query each source
                  |       |
                  |       +-- tasks              (annotation stats)
                  |       +-- annotation_efficiency (efficiency)
                  |       +-- user_activity      (user activity)
                  |       +-- data_sync          (sync status)
                  |       +-- data_lifecycle     (lifecycle stats)
                  |       +-- augmentation       (augment stats)
                  |       +-- quality            (quality reports)
                  |
                  +-- Inject system_prompt (merge or compare mode)
                  |
                  +-- LLM / OpenClaw --> Streaming Response
```

### 数据源注册表（7 个预定义源）

| 数据源 ID                | 分类       | 说明                                |
|:------------------------|:----------|:------------------------------------|
| `tasks`                 | core      | 标注任务统计（总数、状态分布、完成率）   |
| `annotation_efficiency` | analytics | 标注效率（平均时间、通过率、趋势）      |
| `user_activity`         | analytics | 用户活跃度（登录、操作频次）            |
| `data_sync`             | data      | 数据同步状态（源数量、记录数、成功率）   |
| `data_lifecycle`        | data      | 数据流转统计（各阶段数据量）            |
| `augmentation`          | data      | 数据增强统计（任务数、处理量）          |
| `quality`               | analytics | 质量报表（评分分布、趋势）             |

### 管理员配置

管理员通过 AI 集成页面（`/admin/ai-integration` → 数据源权限 Tab）控制：

- 每个数据源的启用/禁用
- 访问模式：只读 (read) 或读写 (read_write)

### 输出模式

| 模式      | 说明                           |
|:---------|:-------------------------------|
| merge    | 合并所有数据源结果，生成综合分析  |
| compare  | 分数据源展示，便于对比分析       |
